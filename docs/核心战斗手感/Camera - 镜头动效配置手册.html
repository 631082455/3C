1. 给技能加镜头(标准流程)

  1. 选资产:从预设库(第 3 节)选一个 CME;库里没有再新建(Content Browser → Data Asset → CameraMotionEffect,命名 CME_ 前缀,放 /Game/Camera/Effects/)。
  2. (可选)调形状:打开资产改数值;要"冲出-回弹"这类节奏,在"波封 → 形状曲线"内联控件里直接拖关键帧。
  3. 挂到技能:打开技能的 HAD → 选触发时点的技能模块数组(激活 / 按下 / 松开 / 确认)→ 加一条相机模块(FSkillModuleCamera)→ AddOrRemove=Add → "镜头动效"数组里加一条:选 CME + 填强度。可同时挂多条叠加。
  4. 完事。技能结束 / 被打断 / 死亡自动淡回,不需要也不存在"清除条件"配置。
另一种挂法——蒙太奇窗口:在蒙太奇上加 ANS_CameraMotionEffect(AnimNotifyState),盖住想要的段(冲刺位移段、前摇蓄力段)。窗口即生命周期,改蒙太奇时长镜头自动跟,打断时引擎保证清理。带时序的效果优先用这种。

2. 字段速查(CME 资产)

分组字段说明
旋转旋转偏移(度) + 旋转空间绕枢轴 Orbit=镜头绕角色转(演绎,≤25°);原地踢 ViewKick=机位不动视线甩一下(命中/开火,≤3°)。Pitch 负=俯视
取景位置偏移枢轴系位移,随控制朝向
臂长增量 + 单位正=拉远负=推近;百分比模式跨体型英雄比例一致
FOV增量 + 单位正=变广角;不影响灵敏度;绝对模式限 ±15
波封混入 / 混出时间必须 >0(命中类 0.02~0.05,大招类 0.2~0.6)
持续时间-1 = 跟随技能生命周期(holding/引导用);正数 = 定时
形状曲线(内联,可选)横轴 0-1 进度,纵轴强度倍率(可 >1 过冲、可负值回弹)。短冲击才画;长状态留空=梯形。峰值越靠左越"脆",负值段越深回弹越明显
滞后位置Lag速度覆盖松跟随卖加速感(冲刺用,3~8)
震屏震屏类 + 强度与动效同生共死,结束必停
策略优先带 / 重触发带按内容选(受击/移动/技能/大招);受击带用"高强度胜出"

使用位点(技能模块 / ANS 里)只填:动效引用、强度(整体缩放)、可选播放速率与时长覆盖。

3. 预设库(/Game/Camera/Effects/)

招牌预设(九成情况选这层,填个强度完事):

CME_HitKick 命中踢 CME_HitKick_Heavy 重击踢(带回弹曲线,可当曲线样例) CME_Dash 冲刺(FOV+拉远,跟随生命周期) CME_Ult_TiltPull 大招俯角+拉远 CME_Cast_LookUp 抬头仰角

单通道原语(要调配比 / 各通道不同节奏时,在使用位点堆多个):

CME_FOVPunch FOV+8 CME_DistPull 拉远+120 CME_KickUp 上踢1.2° CME_OrbitTilt 俯角-10°

新建纪律:先库里选 → 不行就原语堆叠 → 还不行且要单英雄试点就建变体(CMEV,认父只记差异)→ 三条都不合适才新建 CME。库里数值均为起点值,以实调为准。

4. 调试与调参(PIE 控制台)

命令作用
Moba.Camera.Debug 1屏显面板:活跃动效 [资产 | 带 | 权重条 | 强度 | 剩余]。权重条负值(带 <)= 正在反向回弹
Moba.Camera.Effect.Play CME_X 1.0不放技能直接试放(任意目录按名找)
Moba.Camera.Effect.Slomo 0.1全局慢放看清 0.18s 的踢;无参恢复 1.0
Moba.Camera.Effect.Loop CME_X 1.0 1.5每 1.5s 循环重放;LoopStop 停止
Moba.Camera.Effect.ClearAll一键清空全部动效与震屏

实时调参:PIE 开着,改 CME 资产的字段或曲线,下一帧生效——配合 Loop+Slomo 边看边调。存盘走 p4(先 checkout,存完看 LogSavePackage)。

配错会报:存盘时自动校验——混合时间为 0、旋转超预算、变体没父资产等直接拒绝;空动效、移动带超 3 秒等给警告。

镜头心电图(逐帧记录仪):GameProject/Scripts/camera_trace.py。改文件顶部 MONTAGE_NAME(要录的蒙太奇)和 PROJ_CLASS_PATHS(投射物类,没有可留空),编辑器 Python 执行 import camera_trace; camera_trace.start() 挂上;之后 PIE 里每播一次该蒙太奇就自动录一份 CSV 到 Saved/CameraTrace/trace_N.csv(逐帧:蒙太奇进度 / 镜头位置与旋转 / FOV / 臂长 / ControlRotation / 投射物首现标记);camera_trace.stop() 卸载。四类问题它一锤定音:①真实发射帧(别信动画描述——CrossSlash 靠它抓出"逻辑 0.03 发射、动画 0.37 表演"的错位);②时序验证(压缩是否贯穿出手帧、punch 峰值与投射物是否同帧、释放落在恢复期哪段);③准星纯净(ControlRotation 列全程不动 = 动效没污染操作);④静默失配("以为配了其实没写进去"在曲线里立刻现形)。CSV 直接丢给 AI 分析即可,不用自己看数。

5. 常见配法

想要什么怎么配
大招俯角+拉远,结束收回CME_Ult_TiltPull 挂激活时点,强度 1.0。打断/死亡自动收
Holding 期间拉远(如 Kallari Shift)臂长+150、持续=-1 的 CME 挂"进入 holding"时点;释放/取消自动清。要"松手立刻收回"→ 松开时点加一条 AddOrRemove=Remove
冲刺速度感,跟蒙太奇走CME_Dash 挂冲刺蒙太奇位移段的 ANS 窗口——改蒙太奇时长镜头自动跟
命中踢更"脆"打开 CME_HitKick 曲线,峰值关键帧往左拖(t 0.2→0.12);回弹更明显 → 负值段拉深
"俯角减半、拉远不变"这类配比别新建资产:使用位点改挂 CME_OrbitTilt ×0.5 + CME_DistPull ×1.0 两条原语
只给某个英雄试新手感建 CMEV 变体(父=原资产,只填差异倍率),该英雄引用变体;试好把数值搬回父资产、删变体

6. AI 生成(一句话出 CME)

已验证的辅助路径:用一句话描述效果("重锤下砸,带回弹"),AI 推理出字段并落成合法资产,再对话微调("再脆一点")。

  • 怎么用:在连接本项目的 AI 会话(Claude Code 等)里直接描述效果即可;AI 会经 GameProject/Scripts/cme_authoring.py 写盘——该脚本自带合法性钳制(超预算自动缩、缺省自动补),生成的资产必定过校验。
  • 预期:初稿方向对、细节靠你在 PIE 里 Loop+Slomo 看完提修改意见——"生成 → 看 → 说哪不对"循环,不是一次到位的魔法。
  • 生成的资产与手配的完全同构,后续手工改、挂技能、进库都一样。

7. FAQ

问题答案
我的动效没生效?开 Moba.Camera.Debug 1 看面板:没出现=触发点没走到/资产没挂上;出现但 W=0=还在混入或强度为 0;出现后立刻消失=技能瞬间结束了(检查持续时间与挂的时点)
会不会影响子弹/锁定?不会。动效只写画面,弹道/目标选取读的是不含动效的判定视图——这是系统结构保证,不是配置项
两个效果一起触发会怎样?自动叠加(求和),不互相覆盖;叠太多有软上限自动压,不会晕。什么都不用配
旧的 ACD 资产还能用吗?能,照常跑。但新技能一律用 CME;想给老技能加旋转/更干净时再迁
开镜/倍镜怎么配?配不了——那是"换一套持续机位"的状态镜头,本项目已决策不做(见设计稿第 7 节)
形状曲线在哪画?就在 CME 资产"波封 → 形状曲线"那个内联小控件里,点开直接拖点,不需要外部曲线资产
