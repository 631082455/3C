MM 动画数据和自动化工具（for 动画师）

给动画师的工具书。Motion Matching 的片段上要带哪些曲线、每条是干嘛的、缺了会怎样、用哪个工具一键烘、什么时候可以整层不烘。「缺了会怎样」逐条经引擎源码核实。不需要懂 MM 原理，照着做就行。

1工具箱

本管线一个自研工具打底,曲线层用引擎现成件。先认识自研的这个。

mm_rootmotion_baker — 原地动画反烘工具（自研）

移动动画 K 出来都是原地的:位移不用 K,也不该 K（K 了也对不上程序里的移动速度）。但 K 完还剩一套数据要定义:这条动画在游戏里该走多快、root 轨迹长什么样、MM 承重层的四个标记怎么配。以前这套靠人肉配,现在自动化了。

工具用脚锁法直接从你 K 的脚步里把速度和轨迹量出来:触地的脚往后划多快,角色就该往前走多快。烘出 root 轨迹的同时,四个标记（root motion 开关、锁根、Zero 锁、循环）一次配平——这四个标记漏任何一个都是一类线上事故（身体漂移 / 全程鬼畜）。外来资产、动捕素材同理:原地进,带轨迹出。

位置:GameProject/Content/Python/mm_rootmotion_baker.py。编辑器 Python 控制台三条命令,输出都是真实运行记录:

rmb.measure(源片段)只测不改

测这个片段该走多快。烘之前先看一眼,数值过一遍手感。

>>> import mm_rootmotion_baker as rmb >>> rmb.measure('/Game/.../Run_Fwd_C') [RMBaker] Run_Fwd_C 速度=537.9 cm/s 方向=(0.00,1.00) 触地区间=3 个 [RMBaker] ↑ 数值请过一遍手感: 速度感对不对?(工具不会自己报警)
rmb.bake(源, 目标, loop=)产出 *_RM

烘轨迹 + 四标记配平,产出可直接进 MM 库的片段。循环片段匀速保循环,停步片段逐帧积分出减速。

>>> rmb.bake('/Game/.../Walk_Fwd_C', ... '/Game/Heroes/X/Walk_Fwd_C_RM', loop=True) [RMBaker] ✓ Walk_Fwd_C_RM mode=constant loop=True 末位移=(-2,210) 均速=170.6 cm/s [RMBaker] 保存 ...(以日志 LogSavePackage 为准)
rmb.audit(目录)一键体检

扫整个目录:均速、四标记、曲线数。烘完跑一遍,接手别人的资产也先跑一遍。

>>> rmb.audit('/Game/Heroes/MarvelTest/_MMSpike') 片段 均速 标记 曲线数 Run_Fwd_C_RM 566.0 +RM +锁根 +循环 17 Walk_Fwd_C_RM 156.7 +RM +锁根 +循环 2 Idle_C_RM 0.0 -RM +锁根 +循环 2 ←漏配 提醒:均速≈0 的 *_RM = 轨迹没烘上
战绩与边界

漫威角色的 21 个移动片段就是这条工具链烘的（跑 6 向 492~566、走 6 向 152~170、停 6 向减速轨迹）;audit 首跑就抓出 1 处手工漏配的 root motion 开关。

边界:只适用脚步驱动的地面移动;烘出的速度必须人工过手感（工具算错 40% 也不会自己报警,管线史上真错过一次）;批量几十个片段用 rmb.bake_batch,别手点。

2一图看懂：MM 吃两层数据

动画师只需要一个概念：MM 的数据分两层。选片段层决定此刻播哪条动画（靠片段根骨骼的位移轨迹去匹配），修饰曲线层决定播出来精不精（锁脚防滑、切换时脚步对齐）。选片段层缺了直接坏；修饰层缺了只是不够精，而且每条曲线缺失时引擎都会安全降级，不会播出鬼畜。

3曲线图鉴

六条曲线五张卡（FootSpeed 左右一对算一张）。每张卡：人话一句、典型形状、谁在消费、缺了会怎样、用什么烘。徽章含义：近镜头补 按镜头裁决；本项目不读 当前动画蓝图不消费这条曲线，烘了也没人读。

footspeed_l / footspeed_r近镜头补

这只脚此刻移动多快。踩在地上≈0，甩回去时一个峰，左右脚交替。

消费端
FootPlacement（足部放置）节点，用它判断脚踩没踩实：踩实就锁脚、贴地面。
缺了
节点当作「永远没踩地」，整个变直通：不锁脚也不贴地，但不出毛病。远镜头看不出，近镜头有厘米级滑步。
烘焙
AM_FootSpeed_L / _R，一键。默认测 ball_l / ball_r（脚趾球）的移动速度。
反例
整条平 0 = 骨名没指上（见第 4 节第 1 步）。
phase近镜头补

这一帧走到步态循环的百分之几。每一步从 0 爬到 1，再落回 0。

消费端
蓝图侧的混合同步：片段切换时用它对齐左右脚相位，避免左脚接左脚。
缺了
切换瞬间脚步相位可能对不齐，惯性混合会吸掉大半。收益在六条里最低。
烘焙
AM_BakePhaseCurveFromFootstepNotifies。前置：片段上必须先有脚步 notify（第 2 节第 2 步），顺序不能反，反了烘出来是垃圾。
movedata_speed本项目不读

这个片段自己的移动速度。匀速循环是一条平线，起停片段是一条坡。

消费端
步幅/速率缩放的参考值。本项目动画蓝图里速度走的是蓝图值引脚，不读这条曲线。
缺了
StrideWarping 的保护阈值直接回退为不缩放（1.0），安全。
烘焙
暂无必要。未来若改成曲线驱动再议。
enable_orientationwarping本项目不读

开关线：这一段允不允许「下半身朝移动方向扭」。循环段常量 1，特殊段落 0。

消费端
OrientationWarping 的开关。本项目是布尔＋速度方向驱动，不读曲线，所以缺了照样扭、strafe 不受影响（已实测）。
烘焙
AM_OrientationWarpingAlpha，一键。仅在未来改回曲线驱动时需要。
enable_playratewarping本项目不读

开关线：这一段允不允许变速播放来贴合实际移速。形状同上一张，方波。

消费端
播率缩放的开关。本项目播率由蓝图侧驱动，不读曲线；缺了等于不启用播率扭曲，回到原速播放。
烘焙
AM_RateWarpingAlpha，一键。
曲线名大小写不用纠结：工具烘出来叫 FootSpeed_L、在产英雄片段上显示 footspeed_l，引擎里是同一个名字（FName 不分大小写）。
sync marker（脚步同步标记）近镜头补

不是曲线，是打在时间轴上的脚步落地标记。它是 phase 曲线的原料，也是传统同步组对齐脚步的依据。

烘焙
SyncMarkerAnimModifier（AnimationLocomotionLibrary 插件自带）自动侦测落脚打标。
人工位
标完必须肉眼过一遍时间轴：转换来的片段（原地反烘 root motion 那种）自动侦测容易错拍，错拍会顺着毒到 phase。这一步是动画师的眼睛值钱的地方，工具替不了。

4烘焙工作流

五步，有顺序依赖：notify 在前，phase 在后。FootSpeed 与 enable 两类不依赖别人，可以随时烘。

查骨名（只做一次，每个新骨架）

打开目标骨架，确认有没有 ball_l / ball_r。有就直接用；没有（比如叫 toe_l 或别的），把 AM_FootSpeed_L / _R 各复制一份，改 Bone Name 字段指到对应的脚趾球骨。指错或不指的结果是整条曲线平 0。漫威骨架 SK_1043_1043001 实测自带 ball_l / ball_r，直接用原版。

打脚步 notify / sync marker

动画序列编辑器里挂 SyncMarkerAnimModifier，Apply。然后拖时间轴逐个核对落脚点位，错的手动挪。

烘 phase

挂 AM_BakePhaseCurveFromFootstepNotifies，Apply。前一步没做或没核对,这一步产出的就是垃圾。

烘 FootSpeed 与 enable 开关

挂 AM_FootSpeed_L、AM_FootSpeed_R（如第 1 步需要则用复制改骨的版本），按需挂两个 Warping Alpha，Apply。

保存并自查

保存资产，按第 3 节清单逐条过。批量烘（几十个片段）不要手点，走 AI 批量通道（第 5 节）。

挂法：动画序列编辑器 → Window → Animation Data Modifiers → Add Modifier → 选上述资产 → Apply All。Modifier 记录在片段上,改完源动画可以 Reapply 重烘。

5烘完自查

查什么合格长相不合格 = 什么坏了
曲线层
footspeed 形状触地段贴 0，摆动段有峰，L / R 交替错开整条平 0：骨名没指上。峰值贴地不归零：测错了骨或空间
phase 形状每步 0→1 连续爬升、瞬间落回,落点对齐脚步 notify乱跳或平线：notify 缺失或错拍,回第 4 节第 2 步
承重层顺手带一眼（不是曲线,但同一批片段常见坑）
循环标记 loop循环类片段（待机/走/跑）勾上;停步、转身不勾不勾:MM 每次重选都从头混入,全程抽搐
锁根 bForceRootLock原地反烘 root motion 的转换片段必须勾不勾:身体脱离胶囊自己飘出去
速度手感进 PIE 跑一圈,脚下不滑滑步=烘焙速度错了。工具算出来的速度不会自己报警,曾错过 40%(361 → 实际 565),靠人看出来的

6什么时候可以整层不烘

这一节给你省工的权力。修饰曲线层缺失时引擎全部安全降级（已逐条读源码核实,漫威角色 21 个片段零曲线通过验收）,所以烘不烘看镜头,不看惯例。

项目形态裁决
远镜头 ＋ 平地（当前 MOBA 形态）整层不烘。省下的是每个角色一道工序
镜头拉近到看清脚 / 上坡地形 / 第一人称 / 回放模式,任一出现补 FootSpeed 优先（唯一做实事的一层）,sync marker ＋ phase 次之
两条 enable 开关、movedata_speed本项目动画蓝图不消费,烘了也没人读。除非蓝图改回曲线驱动,否则任何形态都不用烘

一句话记住：选片段层永远必做,修饰层按镜头裁决。拿不准就先不烘,缺失是安全的,补烘随时可以（Modifier 一键）。
