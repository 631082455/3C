概述

口径更新（2026-07）：本篇为早期接入口径，两处已被后续实现取代—— ① Steel 的 mantle_height_from_montage=120 为旧单值口径，现行为「动作 × 高度」四格补偿（Steel 当前仅 Climb Tall = 30cm），见《多体型适配实录》； ② "Mesh 必须打 Tag 才会重定向"来自旧注释，现行主链路是 HeroData.IkRetargeterAsset 由 C++ 直接注入，见《新增英雄移动动画配置指南》。正文对应处已加标注，其余步骤仍适用。

一、系统概述

项目中存在两套翻越系统，由 MCD_[CharName] → mantle_setting.use_new_traversal_system 控制：

字段值 使用的 Ability 特点
false GA_Mantle（老系统） 单一 Montage，仅支持 Climb/Mantle；有已知结束 snap 问题（已修复）
true GA_Traverse（新系统） 四类型（Hurdle / Vault / Mantle / Climb）+ Catch；完整 Motion Warping

强烈推荐所有新角色使用新系统（true）。

二、方向 A：常规体型（骨骼与 UEFN Mannequin 接近）

适用角色举例：TwinBlast、Sparrow（切换新系统后）

2.1 动作重定向

常规体型直接复用 GASPALS UEFN Mannequin 动画库，通过运行时自动重定向驱动：

  1. 创建 IK Rig：IK_[CharName] - 参考 IK Rig：/GASPALS/Characters/UEFN_Mannequin/Rigs/IK_UEFN_Mannequin - 为目标骨骼创建对应的 IK Chains

  2. 创建 IK Retargeter：RTG_UEFN_to_[CharName]Auto - Source：IK_UEFN_Mannequin → Target：IK_[CharName] - 在 FK Chains 阶段调整 chain offsets 直到 T-Pose 对齐

  3. 配置 HeroData（HeroData_[CharName]）：

字段 值
HeroAnimClass ABP_HeroCharacter_MM（或角色专属 ABP）
RetargetMesh SKM_UEFN_Mannequin
RetargetAnimClass ABP_GenericRetarget
IkRetargeterAsset RTG_UEFN_to_[CharName]Auto

重要：RetargetMesh 和 RetargetAnimClass 必须同时配置，缺一不可。 旧口径提醒：ABP_GenericRetarget 内部注释 IF THERE IS NO TAG, THE CHARACTER WILL NOT RETARGET!!！ 描述的 Tag + Map 路径已不是现行主链路——当前由 HeroData.IkRetargeterAsset 经 C++ 直接注入，配齐 HeroData 三字段即可，无需依赖 Mesh Tag（详见《新增英雄移动动画配置指南》第 7 节）。

2.3 AnimMontage Asset User Data

每个翻越 Montage 上必须配置 Ability Montage Data（AbilityTag），否则 GAS 无法识别该 Montage 属于哪个 Ability。

  • 在 Montage 的 Asset User Data 里添加 UAbilityMontageData
  • 设置对应的 AbilityTag（如 GA.Traverse.Climb）

三、方向 B：体型差异巨大（骨骼比例与 UEFN Mannequin 差异显著）

适用角色举例：Steel（矮粗型）、体型特殊的机械类角色

3.1 特殊问题

体型差异大的角色，直接使用标准 IK Retargeter 会出现：

  • 走路抖动/闪烁：手脚 IK 目标跟随 Mannequin 位置，与角色骨骼不匹配
  • 跳跃手臂扭曲/拉伸：FBIK solver 在跳跃姿态下强行将手拉向 Mannequin 手的世界坐标

3.2 IK Rig 配置（IK_[CharName]_new）

在 FBIK Solver（Run IK Rig 阶段）中：

手部 Goal（LeftHandIK / RightHandIK）：必须从 solver 断开

# 使用 Python 脚本操作
ctrl = unreal.load_asset('/Game/.../IK_[CharName]_new')
with unreal.ScopedEditorTransaction('Disconnect Hand IK') as trans:
    ctrl.disconnect_goal_from_solver('LeftHandIK', 0)
    ctrl.disconnect_goal_from_solver('RightHandIK', 0）

断开后手臂为纯 FK，完全跟随重定向骨骼，消除跳跃扭曲。 副作用：翻越时手不再贴附表面，可接受（动画本身已有正确的手位置）。

脚部 Goal（LeftFootIK / RightFootIK）：调整参数

参数 推荐值 说明
bone foot_l / foot_r 目标骨骼改为 foot，而非 ball
chain_depth 2 影响范围限制在 2 段骨骼（foot + lower leg）
strength_alpha 1.0 完整 IK 强度

Solver 全局参数

参数 推荐值
root_behavior PIN_TO_INPUT
iterations 10
sub_iterations 20
mass_multiplier 0.5

3.3 FK Chains 阶段：禁用 ik_foot_root

ik_foot_root 是一个专属于大体型角色可能存在的额外链，会干扰 FK 重定向。

在 IK Retargeter（RTG_UEFN_to_[CharName]Auto）的 FK Chains（阶段 [1]） 中：

  • 找到 ik_foot_root chain
  • 将其 Enable 设为 false
# Python 操作示例
retargeter = unreal.load_asset('/Game/.../RTG_UEFN_to_[CharName]Auto')
controller = unreal.IKRetargeterController.get_controller(retargeter)
with unreal.ScopedEditorTransaction('Disable ik_foot_root') as trans:
    controller.set_enabled_chain('ik_foot_root', False）

不禁用此 chain 会导致走路时脚部抖动/闪烁。

3.4 MotionWarp SkewWarp 参数

在翻越 Montage 的 MotionWarping Notify State 里：

参数 常规体型 大体型注意事项
warp_to_feet_location true 大体型必须为 true，否则 root 偏差大
add_translation_easing_func LINEAR 建议 LINEAR，网络下更稳定
warp_target_name FrontLedge（新系统）/ MantleTarget（老系统） 不可改动

3.5 MCD 额外参数

大体型角色的 mantle_height_from_montage 需要根据角色实际骨骼高度调整：

  • 含义：手搭到上沿后，root motion 还需向上移动多少 cm 才能站上去
  • Steel 的值：120（悬挂风格动画）——旧口径：现行 Steel 已改用四格补偿（当前仅 ClimbZOffset_Tall = 30cm，mantle_height_from_montage=-1 仅作非 Mantle/Climb 回退），以《多体型适配实录》为准
  • GASPALS 风格动画：0（sync point 已是最终站立位置）

四、C++ 层已知修复

4.1 老系统（GA_Mantle）结束 snap 修复

文件：Source/GameMoba/AbilitySystem/Abilities/HeroAbility_Mantle.cpp

修复内容：

  1. EndAbility：MOVE_Falling → MOVE_Walking （Climb 完成后角色已在 ledge 上，用 Falling 会触发落地纠正动作）

  2. ActivateAbility / EndAbility：增加网络纠正抑制 ```cpp // ActivateAbility 中 HeroMovement->bIgnoreClientMovementErrorChecksAndCorrection = true; HeroMovement->bServerAcceptClientAuthoritativePosition = true；

// EndAbility 中 HeroMovement->bIgnoreClientMovementErrorChecksAndCorrection = false; HeroMovement->bServerAcceptClientAuthoritativePosition = false; ```

即使使用老系统，以上修复也是必要的。

4.2 Warp Target 计算逻辑

文件：Source/GameMoba/Character/HeroCharacterMovementComponent.cpp 函数：GetMantleStartLocation(）

WarpTarget.Z = Surface_Hit.Z - MantleHeightFromMontage
WarpTarget.XY = Surface_Hit.XY - WallNormal.XY * CapsuleRadius
  • 新系统 Warp Target 名：FrontLedge（在 StartMantle(） 中设置）
  • 老系统 Warp Target 名：MantleTarget（在 MantleCueNotify_Static GC 中设置）

五、快速检查清单

新角色接入前确认

  • [ ] HeroData_[CharName] 的 RetargetMesh / RetargetAnimClass / IkRetargeterAsset 均已配置
  • [ ] Mesh 上有正确的 Mesh Tag（ABP_GenericRetarget 依赖此 Tag 选择 Retargeter）
  • [ ] MCD_[CharName] 的 use_new_traversal_system = true
  • [ ] mantle_height_from_montage = 0（GASPALS 动画）或正确的悬挂高度（专属动画）
  • [ ] 34 个翻越 Montage 字段全部填写，无遗漏
  • [ ] 每个 Montage 的 Asset User Data 中有 Ability Montage Data + AbilityTag

大体型额外确认

  • [ ] IK Rig 中 LeftHandIK / RightHandIK 已从 FBIK solver 断开
  • [ ] IK Rig 中脚部 Goal 的 bone 为 foot_l / foot_r，chain_depth = 2
  • [ ] IK Retargeter FK Chains 中 ik_foot_root 已 disable
  • [ ] Solver root_behavior = PIN_TO_INPUT

运行时验证

  • [ ] 单机：无攀爬结束 snap
  • [ ] 联网：无联网抖动（服务端纠正已抑制）
  • [ ] 跳跃：手臂无扭曲/拉伸
  • [ ] 走路：无脚部抖动/闪烁
