Traversal Animation Framework

攀爬 / 翻越动作生产视角

这份文档只讲一个结论：攀爬动画先补齐场景覆盖，再做细节变体。当前生产范围先收敛到 Hurdle / Vault / Mantle / Climb 四类，动画生产先保证每一类都有稳定可用的动作。

当前生产决策：只保留 Run 版本作为源资产；低速、贴墙、原地触发不单独列动画资产；Walk / Stand 档和左右脚拆分都不作为基础库必需项。

03 / Shared Humanoid Framework

给动画同事的三个结论

讨论时优先对齐这三件事，不需要先陷进 Trace 参数或程序实现。

先补齐动作类型

缺某一类动作，会导致对应场景完全没有合适表现；同一类里做多少细分变体，可以放到后面。

  • 先保证 Hurdle / Vault / Mantle / Climb 都有可用动作。
  • Catch 先不进入本轮生产和数量统计。
  • 再考虑同一动作类型里的风格、节奏和细节丰富度。
  • 覆盖面比同类变体数量更重要。

基础动作先通用

基础动作只解决身体怎么通过障碍：Root 路径、手脚接触、落点和结束姿态。角色性格、武器差异后面再叠。

  • 不要每个角色重做一整套攀爬。
  • 先把通用人形动作跑通。
  • 角色差异用 Overlay、Additive 或少量补丁处理。
04 / Run-Only Decision

资产收敛决策：只保留 Run 源资产

Walk / Stand / 低速入口 / 左右脚相位都不再作为独立生产维度。所有攀爬翻越动作只生产 Run 版本；贴墙、低速、原地触发由系统接入侧用同一 Run 动作处理，不在动画表里单列资产。

Video 攀爬动作三挡对比：源资产收敛依据

该视频用于对比不同入口档位的动作差异。后续生产不再按 Stand / Walk / Run 三套源资产扩展，也不单列低速、贴墙、原地入口；统一以 Run 版本作为动作母版。

决策项 做法 原因
只生产 Run 源资产 Hurdle / Vault / Mantle / Climb 都以 Run 版本为基准，优先保证动作完整度、Root Motion 路径和手脚接触质量。 Run 版本包含最完整的速度、发力和身体通过障碍信息。用它作为源资产，可以让通用库的动作质量集中在一套核心数据上，而不是被 Stand / Walk / Run 三档稀释。
删除独立 Walk 档 Walk / Stand 不再要求单独生产。中间速度由系统侧播放速率、Blend 和 Warp 处理。 Walk 档资产量增加明显，但体验收益不稳定。竞技化移动更重视输入响应和路径稳定，Walk 独立档容易变成维护成本，而不是核心手感收益。
低速 / 原地不单列资产 不再为贴墙、低速、原地入口建立独立动画条目；统一使用对应 Run 动作。 这类入口是触发条件差异，不是新的动画生产族。先保证 Run 母版质量，系统侧处理触发时机和对齐窗口。
重新配置动画事件 同一 Run 动作接入不同触发场景时，需要在系统侧重配 Montage 起点、Blend In、Motion Warping 窗口、PlayMontageNotifyWindow 和 BlendOut 时机。 如果 Notify 和 Warp 窗口仍按单一场景走，会出现第一帧拉扯、抓边错位、穿墙或结束落点不稳。
这不是“用 Run 动画硬套所有速度”，而是“把 Run 当作高质量动作母版”。接入侧可以根据触发条件选择不同播放起点或窗口，但这些不作为动画同事的新增交付项。
05 / Classification

系统分类规则对应的动画职责

下面不是让动画师背参数，而是说明每种关卡几何为什么需要不同动作族。动作生产时要保证这些几何差异在身体路径、手脚接触和结束姿态上能被看出来。

动作覆盖原则：先保证 Hurdle / Vault / Mantle / Climb 四类场景都有稳定动作；同一类里有几个小变体、表现多细，放在覆盖面成立后再补。Catch 本轮暂缓，不进入数量统计。
动作族 典型触发 动画承担的核心作用 生产时最容易出问题的点
Hurdle翻越低障碍 薄障碍，背面存在落点，并且背面有明显落差。 让角色快速越过低障碍，保持向前动量，翻越后自然落向 BackFloor。 Root 推进不足会像原地跨栏；推进过强会穿帮。落地脚、骨盆高度和胶囊体过障碍的时间要匹配。
Vault越过无背面地面 薄障碍，但背面没有可站立地面，翻过去后进入空中或落差段。 表现“越过边缘后继续下落”的动作，不要求结束在障碍顶部稳定站住。 结束姿态不能强行摆成站立；需要给 InAir / Land 留出自然接续，避免落点前后速度断裂。
Mantle攀上平台 厚平台、台阶或低于 Climb 的可站立顶面。 把角色从障碍前缘拉到顶面，结束时稳定回到 Grounded / MM 移动。 手撑、膝盖、脚掌和顶面高度需要可信；结束帧不稳会导致回到移动循环时滑脚或身体弹跳。
Climb高平台攀爬 高于 Mantle 上限的高平台，当前抽象约 160-360cm。 表现完整的高墙攀爬过程：抓边、蹬墙、抬身、上台、站稳。 高墙动作对体型和比例敏感；手点、脚点、骨盆清障和最后站起速度都需要稳定。
Catch下落抓边 角色处于 Falling，检测到可抓边缘，并满足下落速度窗口。 从下落姿态接入抓边或翻上动作，避免“明明碰到边却掉下去”的负反馈。 第一帧最关键；如果手没有及时对上边缘，会被感知为吸附或拉扯。Catch 的 Warp 目标也必须比普通 Mantle 更保守。

Traversal 动画资产职责表

当前资产以 Run 源资产为主。历史资产里存在 Stand / Walk / Run 多档，但后续生产只保留 Run 动作清单；Catch 暂不进入本轮生产口径。

P0 Climb高台攀爬 Run 单版不单列贴墙资产 Run 源资产覆盖跑动高攀；贴墙、低速、原地触发不新增动画条目，由系统侧接入同一 Run 动作。 高墙最容易暴露身高比例问题。需要明确手抓点、脚蹬点、骨盆清障、上台后的站稳姿态，以及 Tall / Short 体型偏移兼容。
P0 Hurdle快速翻越低障碍 Run 单版必要时再补 V2 Run 源资产覆盖主要翻越体验；低速和原地不作为独立 Hurdle 生产目标。 这是探索和战斗移动中最容易频繁触发的“不断速”动作。重点是快速、低打断感、落地动量连续。低速贴近障碍时，如果 Hurdle 表现不自然，可以由系统降级到 Mantle / Step-up。
P1 Vault越过边缘并进入空中 Run 单版不单列低速资产 Run 版本覆盖主要 Vault。低速或原地 Vault 不作为独立表现目标，必要时由动作分类降级处理。 Vault 的价值在于处理“翻过去不是平台”的场景。动画结束不能假设立刻站住，要给 InAir、落地和速度恢复留余量。没有速度时强行 Vault 的表现风险高。
暂缓 Catch下落抓边覆盖入口 不计入本轮不生产 Catch Hurdle / Vault / Mantle / Cliff，高中低高度优先，入口速度不再作为核心生产维度。 Catch 是容错体验，不是单纯扩充动作数量。重点是下落第一帧抓边可信、Warp 距离短、接入后不拉胶囊体。
P2 角色差异层Overlay / Additive / 补丁 按角色补充 武器、体型、性格、职业动作差异。 不建议为每个角色复制整套 Traversal。优先补上半身保护姿态、手臂风格、面向武器的局部差异和极端体型的偏移参数。
07 / Hurdle Details

Hurdle：低障碍快速翻越

Hurdle 的第一目标是“不断速地跨过去”。它不应该看起来像一次完整攀爬，也不应该让玩家感觉角色被动画长时间扣住。

09 / Climb Details

Climb：高平台攀爬

Climb 处理 Mantle 无法可信覆盖的高平台。它是最容易暴露体型、墙面高度和手脚接触问题的动作族，因此本轮只集中打磨 Run 母版。

Run Climb跑动高攀 1基础单版 从现有 Run Climb 中选一个稳定版本作为母版。 高速接近高平台时进入高攀，尽量减少停顿感。 起跳、抓边和身体上提要形成连续力线。不要让角色先停住再爬，除非故意做重型表现。
Climb 不建议只靠播放速度缩放去兼容所有高度。当前系统已按高度和体型做起始位置偏移，动画侧仍要保证关键接触点可以被 Motion Warping 稳定拉到目标，而不是依赖过大的拉伸补偿。

12 / Motion Warping Contract

动画交付约束：Root Motion、Warp 目标和结束姿态

Traversal 的动画资产不是单独看的表演片段，而是和胶囊体、碰撞、Warp 目标一起工作的动作。下面这些约束要作为生产验收项。

约束项 具体要求 为什么重要
Root Motion 方向 Root 推进应和动作族一致：Hurdle / Vault 向前穿过，Mantle / Climb 向前上台。 Root 与胶囊体运动不一致时，会出现滑脚、穿墙、落点偏差或被 Motion Warping 强拉。
FrontLedge 手抓边、胸口靠近墙面、上台前缘都要围绕 FrontLedge 成立。 这是 Mantle / Climb 最主要的对齐点。接触点不可信会直接破坏攀爬真实感。
BackLedge / BackFloor Hurdle / Vault 要明确越过后缘和落地目标，动作结束不能停在障碍中间。 这两类动作通常会临时关闭障碍碰撞，如果动画没有稳定越过后缘，就会被看成穿帮。
脚相位 基础库默认不拆左右脚。只有出现明显反脚、错脚或第一帧穿帮时，再补脚相位变体。 左右脚拆分主要提升衔接细腻度，但会放大资产量。当前项目优先保证动作类型覆盖和稳定可用。
结束姿态 Mantle / Climb 结束要能回 Grounded；Hurdle / Vault 结束要能回 InAir 或 Land。 很多问题不在动作中段，而在结束后的第一帧。结束姿态决定能否平滑接回 MM。
体型兼容 通用库先服务标准人形。高矮、壮瘦角色通过 Z Offset、胶囊体、局部补丁和少量专属动作处理。 如果每个角色都重新做全套资产，维护成本会失控；但完全不管体型又会在 Climb 上暴露严重穿帮。
13 / Naming & Delivery

建议的动画命名与交付信息

命名不是形式问题，它直接影响后续批量检索、资产合并、英雄复用和问题定位。后续命名只保留 Run 源资产口径，不再扩展 Walk / Stand / 低速入口。

推荐命名字段

角色 / 姿态 / Traversal / 动作族 / 高度层 / 入口来源 / 方向

  • 动作族：Hurdle、Vault、Mantle、Climb。
  • 入口来源：RunSource。
  • 脚相位：不作为基础库必需字段，只有出现明显接入问题时再补。
  • 高度层：Low、Med、High 或项目现有 1_0、2_5。

每个动作交付时应标注

  • 适用动作族和入口来源，不要只给“好看的翻越动画”。
  • 是否需要 FrontLedge、BackLedge、BackFloor。
  • 入口和结束应该接 Grounded 还是 InAir。
  • Montage 起点、Blend In、Warp 窗口和 BlendOut 是否已经重新配置。
  • 是否存在明显反脚、错脚或第一帧穿帮；没有问题则默认合并。
  • 是否适合所有人形，还是只适合某个体型或武器。
