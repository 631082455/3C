移动优化二期：起停与折返手感

一期把数据流跑通之后，角色能动了，但手感是生肉：起步软、起步动画播一半被抢，松键停不干脆，反打方向掉头拖沓，移动中脚下还晃。二期集中在 3 月上旬，七个提交把"起、停、折返"这条瞬时响应线从能动调到跟手。本篇按问题逐条记录现象、根因、方案与效果，失败尝试保留。

定位移动手感优化六期中的第二期：起步、停步、折返、移动稳定性。竞技移动的胜负手在瞬时动作，这一期就是给三个瞬间立规矩。上承一期数据流接入，下启四期竞技化（二期遗留的起步问题在 6 月复发升级，见四期）。
周期2026-02-28 ~ 03-16，密集期为 03-03 一天四连击（起步保护 → 强制冲刺起步 → 动作裁剪 → 起步冲量）。
成果移动晃动消除、起步动画完整且有蹬地感、松键急停干脆；移速控制归位到移动配置层（MCD），动画层不再越权。
遗留起步方案是"保护 + 裁剪"的表层修法，速度失配的深层根因当时未触及，6 月以 Sprint 战役形式回归（四期）。

总览

总览与阶段索引

项目是 UE5 多人 MOBA 射击，移动层用 GASPALS Motion Matching，一期（1 月）刚把项目自持的数据流搭起来。二期的目标用一句话说：把"能动"调成"跟手"。竞技移动的胜负全在瞬时动作，三问：按下去多快有反应（起步）、松开键多快站定（急停）、反打方向多快掉头（折返）。走位、卡位、骗技能全靠这三个瞬间，当时三问全都不及格。

二期的取向

起停手感的仲裁权分层：速度归移动组件（MCD 配置），动画层只做表现，不许反向控制移动。这一期两头动刀：把 ABP 里越权改移速的逻辑断掉，把起步的表现问题在动画数据与选择层内解决。这条边界后来贯穿了整个系统（四期的 Gait 驱动、六期的方向衰减都沿用它）。

移动稳定性

1.1 移动中角色整体晃动（03-03）

现象：普通跑动中角色躯干持续小幅晃动，视角拉近尤其明显，像脚下不稳。这是二期开工的第一刀，因为它污染所有后续手感判断：起停调得再好，晃着也是废的。

根因：定位到 FootPlacement（脚部落地对齐节点）与 MM 输出的相互作用。MM 每帧输出的姿势本身连续，FootPlacement 在其上再做脚部与地面的对齐修正，两层修正叠加后根链出现高频小幅拉扯。项目烘的移动动画不带 GASP 的 footspeed 曲线（后来 6 月源码核实：缺曲线时该节点按"脚未踩实"的保守路径工作），修正量比 GASP 原生资产更抖。

方案：ABP 里调整 FootPlacement 链路（配套动了 MCD 与 Chooser 各一处），以稳定优先收敛修正幅度。MOBA 拉远视角下，脚部对齐的收益本来就小于躯干稳定的收益，取舍不难。

效果：晃动消除，Steel 移动整体顺了一档。✅ 同 CL 顺手做了 Steel 移动的整体参数过一遍。

1.2 动画层越权：断掉移速控制（03-12）与卡顿修复（03-16）

现象：调参时发现移速的实际生效值和 MCD 配置对不上，追下去发现 ABP 里有逻辑在反向改移动速度；另有场景里移动一顿一顿。

方案：把 ABP 里的移速控制整条断掉，速度仲裁权收归 MCD（移动配置资产）；卡顿按 MCD 参数修（两名英雄一起过），顺带修了一个翻越蒙太奇的衔接。

效果：从此"改移速只有一个入口"。这条架构边界是二期比任何单点修复都值钱的产出。✅

起步战役（03-03，一天四连击）

起步是竞技移动的第一印象。当时的起步四个病并发，一天之内按依赖顺序连修四刀，顺序本身就是方法：先保住动画能播完，再管选没选对，再修动画本身，最后补爆发感。

2.1 起步动画刚起就被抢（起步强制保护时间）

现象：起步动画只播零点几秒就被 MM 重选切走，起步永远播不完整，视觉上"起步没有起步样"。

根因：起步瞬间速度快速爬升，查询特征逐帧变化剧烈，每帧搜索都可能找到 cost 更低的新候选，起步段在连续重选下被切碎。这是 MM 的结构性倾向，不是数据错误。

方案：ABP 侧给起步状态加强制保护时间：起步开始后的保护窗内不接受重选（GASP 的 ForceStartTimer 机制，窗口 1.3s，该值沿用至今）。

效果：起步动画完整播出。✅ 代价是保护窗内响应折返会迟一拍，属于已知取舍，折返手感由折返库自己的动作弥补。

2.2 高速档起步不走冲刺起步（强制适用 Sprint Start）

现象：移速拉到冲刺档起步，播的还是普通跑步起步，脚步频率和位移对不上，起步显"软"。

方案：在起步库条目上做强制适配，保证高速档起步命中冲刺起步动画。

效果：当时症状消失。✅ 但要诚实记一笔：这是条目层的表层修法，"为什么高速起步匹配不到冲刺起步"的深层原因（角色移速与动画原生速度失配、起步库覆盖残缺、Gait 无驱动）当时都没触及。6 月这批根因全数爆发，升级成四期的 Sprint 战役才真正治根。二期这一刀的价值是先让表现及格。

2.3 起步动画头部拖沓（起步动作裁剪）

现象：保护 + 强制适配后起步能完整播了，新问题显形：冲刺起步动画头部有一段蓄力预备，按键到出脚之间隔着可感知的延迟。竞技游戏里这段"演技"是负资产。

方案：直接裁动画：冲刺起步 F 向左右脚两条动画各裁掉头部预备段，按键即出脚。

效果：起步响应立竿见影。✅ 这也是二期最"策划化"的一刀：不动系统，动数据。

2.4 起步缺爆发感（起步微弱冲量）

现象：起步动画对了、完整了、出脚快了，但体感仍然"温"：速度从 0 线性爬升，缺少竞技游戏起步那一下蹬地的冲劲。

方案：起步瞬间给一个微弱的速度冲量（ABP 侧配合过渡数据与 Chooser 调整）。幅度取"微弱"：只补最初几帧的加速度观感，不改变到达满速的总时间，不影响数值层的移速公平。

效果：起步有了蹬地感。✅ 这个思路后来在 C++ 层重生为 StartBoost（FStartBoostSetting，六期遗留里因网络同步问题未启用），二期的 ABP 版是它的前身。

停步与折返（03-04）

3.1 松键滑行，停步动画不出（mm 恢复停步）

现象：松开方向键，角色带着惯性滑一段才站住，停步动画不触发或触发了也接不顺。急停是 MOBA 走位的高频动作（卡位、骗技能、贴反向），这里软了整个走位手感都软。

根因：停步是"移动配置的减速参数 + 停步动画库路由"的双人舞：减速太慢则查询轨迹迟迟不归零、停步动画迟迟不被匹配；路由不对则匹配了也进不了停步库。当时两头都有问题。

方案：MCD 侧调减速（两名英雄一起）、ABP 与 Chooser 侧恢复停步库的路由链，让"松键 → 轨迹预测归零 → 停步动画命中"这条链重新通。

效果：松键即出停步动作，滑行收束干脆。✅

停步这类问题的通用教训：MM 系统里"某类动画不出"优先怀疑路由与驱动变量，其次才是动画数据本身。停步库好好躺在那里，是路上的链断了。这条经验四期排查 Sprint 起步时直接复用（那次也是路由行序的锅）。

3.2 折返：反打方向的掉头（与停步同批打通）

定位：折返（跑动中反打方向的 180° 掉头）和停步共用同一条判定链：松键是"预测轨迹归零 → 停步库"，反打是"预测轨迹反向 → 折返库（Pivots）"，路由都走同一套起停状态。停步链路修通的同一批，折返的命中链路一并验证打通：反打能稳定进折返库出掉头动作，而不是靠巡航动画硬拧。

爆发感：折返和起步是同一族问题的两面，都是"速度矢量突变的瞬间要给玩家一个干脆的确认"。起步冲量（2.4）立的思路天然覆盖折返，这条思路后来在 C++ 化为 StartBoost 时被正式做成双触发：静止起步触发，以及速度与输入方向反向的折返瞬间触发（按方向点积判定）。

如实记录边界：二期对折返做到"链路通、有动作、思路立住"，没有做专项深调。折返库的选择质量（GASP 的 Run 折返库 34 条覆盖全向，Sprint 折返库只有 4 条）这类深水区，四期 Sprint 战役时才暴露出来；折返爆发的网络化（StartBoost 折返触发）挂在六期遗留。

收尾

二期后的状态与遗留

二期收工时的三问答卷：起步即出脚且动画完整可信（裁剪 + 冲量 + 保护窗）；松键即停（减速 + 停步路由）；反打进折返库有掉头动作（链路与停步同批打通）。移动稳定性上，晃动清零，移速仲裁权收归 MCD 单一入口。

遗留（如实）

  • 起步的修法是表层的。保护窗、强制适配、裁剪都在"让表现及格"，速度失配与库覆盖残缺这两个深层根因没动。6 月它们以更复杂的形态回归（冲刺档打摆子、起步误选反身动画），升级成四期的 Sprint 战役。回头看二期不亏：当时的认知和工具（还没有 Rewind Debugger 的 cost 拆解习惯）不足以打穿根因，先表后里是当时资源下的正确排序。
  • 起步保护窗 1.3s 与折返响应存在天然张力，保护窗内折返迟一拍，当时接受，未再调。
  • 折返只做到"链路通、思路立"，未做专项深调：折返库的选择质量（尤其 Sprint 折返库仅 4 条的覆盖残缺）与反打爆发的手感细调都留给了后期。
  • 起步冲量是 ABP 层实现，多人网络下的预测一致性当时未验证；后来 C++ 化为 StartBoost 时发现了同步问题（缺移动合并比较），至今未启用，挂在六期遗留。
