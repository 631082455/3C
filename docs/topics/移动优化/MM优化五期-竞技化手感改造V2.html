定位：重新定义 MM 在竞技移动里的职责

竞技化移动的核心不是单纯提高加速度或最大速度，而是让玩家每次输入都能被系统尽早承认。Motion Matching 仍然负责连续、自然和姿态可信，但不能让动画惯性压过操作意图。

输入意图优先

起步、折返、急停、转镜头时，系统先确认玩家想做什么，再决定用哪段动画表现，而不是让当前姿态惯性决定一切。

真实位移可信

胶囊体的真实移动必须和 Mesh 表现保持一致。移动可以更利落，但不能出现角色被服务器修正、动画被胶囊体拖走的拉扯感。

表现不破碎

竞技手感不是牺牲动画质量。调优目标是在响应变快的同时保留重心变化、脚步方向、转身节奏和起停动作辨识度。

一句话定位：把 Motion Matching 从“优先展示自然运动片段”的系统，调成“以输入意图驱动动画选择，并用后处理保证运动可信”的竞技移动框架。 02 / Architecture 实现拆解

三层拆解：CMC、MM 选帧与动画后处理

移动手感问题很容易被描述成“慢”“飘”“不跟手”，但落地时需要拆到具体系统。我的调优方式是先判断问题属于真实运动、动画选择还是表现修正，再决定该改移动参数、PoseSearch 权重、动画数据还是 ABP 后处理。

运行时链路

  • 输入层：读取移动输入、镜头方向、上帧速度、当前加速度，判断起步、停止、折返和持续移动。
  • CMC 层：负责真实速度、加速度、制动、摩擦、最大速度和网络预测，是玩家实际位移的权威基础。
  • MM 层：根据当前 Pose、Velocity、Trajectory、Facing 等特征在数据库里选最合适的下一帧。
  • ABP 后处理：通过 RootYawOffset、Orientation Warping、Stride Warping、Turn In Place 等修正表现。

策划调优抓手

  • 移动配置：最大速度、加速度、制动、摩擦、起步 / 折返信号窗口、Boost 持续与取消规则。
  • PoseSearch 权重：当前姿态、未来轨迹、速度方向、朝向、脚步相位、连续性惩罚。
  • 动画资产：Start / Stop / Pivot / Loop 的覆盖范围、速度档、角度档和不良片段剔除。
  • 验收 Metric：输入到位移响应、折返角度、Pivot 命中率、停止距离、滑步量、镜头转身体感。
层级负责什么不该负责什么典型问题
CMC 真实移动、预测同步、速度和制动 不负责生成 Pivot 动画表现 速度改快后网络修正、胶囊体和 Mesh 拉扯
Motion Matching 选择最符合当前运动意图的动画片段 不负责替代所有运动规则 起步选慢、折返不进 Pivot、停止仍选 Loop
ABP 后处理 修正朝向、步幅、根骨骼偏移和转身表现 不应该掩盖错误的选帧和移动参数 RootYawOffset 回正慢、Warping 过度、转身发飘
动画数据 提供足够覆盖的 Start / Pivot / Stop / Loop 资源 不应该用后处理硬补缺失动作 角度档缺失、速度档不齐、脚步相位不稳定
03 / Tuning Records 调优问题解决记录

问题现象 → 处理结论：移动专项复盘

条目中的「实例」均取自本系列一～四期与相关案例的实测记录；个别标注 ※ 的为示意值，非实测。

以下记录按工程调试口径整理：现象、定位、处理、验证口径。重点体现 3C 策划如何把移动体验问题拆到 CMC、PoseSearch、ABP 后处理、动画资产和网络预测等具体环节，再收敛成可复用的调优结论。

证据口径先说明：除「实例」标注的条目取自实测记录外，各条「处理 / 验证口径」是调优方向与验收标准的整理，不代表已在本项目逐条落地实测；已完成与未完成的边界以四期、六期的验收实录与遗留清单为准（例如 StartBoost 的 SavedMove 合并补全仍挂在六期遗留，未启用）。与《MM 调参手册》第 09 节症状反查表同题的条目，参数细节以手册为准，本篇保留决策背景与实例。

MM-01

静止起步响应滞后，Start 片段进入过晚

Start Response 现象

从静止到移动时，CharacterMovement 已产生加速度，但 MM 仍维持站立附近 Pose，Start 动画进入滞后。

定位

PoseSearch 对当前站立姿态连续性保护过高，未来轨迹和输入方向权重不足；部分 Start 片段有效起跑帧偏后。

处理

建立起步响应窗口，提高输入方向、预测轨迹和加速度特征权重；让 ABP 感知 StartBoostAlpha，剔除有效位移过晚的起步片段。

验证

静止起步首帧即出现明确移动意图，Mesh 与胶囊体加速趋势一致，保留脚步启动和重心转移。

MM-02

实例：本系列二期的同源病例——高速起步匹配不到冲刺起步，根因是角色移速与动画原生速度失配（后经脚锁法反烘校准，动画原生速度 361 → 565 cm/s）、起步库覆盖残缺、Gait 无驱动三件事叠加。二期的"保护 + 裁剪"只救了表现，根治发生在竞技化战役（四期 V1）。

180 度折返经常绕弧线，不稳定进入 Pivot

Pivot Selection 现象

高速跑动时反打方向，角色有时能做 Pivot，有时只是大弧线转向；竞技场景下反馈不稳定。

定位

Pivot 判定只看输入角度不够，需要结合速度、上一帧移动方向、减速状态和最小触发窗口；MM 成本仍偏向连续跑步。

处理

增加折返角度与速度阈值，加入短暂 Pivot 锁定窗口；提高反向轨迹权重，降低当前 Pose 对折返的过度保护。

结果

120-180 度变向更稳定触发 Pivot，小角度转向仍保持自然过渡。

MM-03

实例：GASP 的 Run 折返库 34 条覆盖全向，而 Sprint 折返库只有 4 条——高速折返"绕弧"很大程度是库覆盖残缺，系统只能拿近似片段凑。补库之前，任何连续性成本调参都治不了根。

Pivot 误触发：小幅方向修正被识别为大角度折返

False Positive 现象

提升折返响应后，玩家微调方向或贴边移动时也容易打出 Pivot，角色表现过重。

定位

缺少输入稳定时间和最小位移速度限制，方向变化一过阈值就立即承认为折返。

处理

增加方向保持帧数、最小速度、触发冷却和小角度死区；把“强 Pivot”和“快速转向”分成两档。

验证

ADAD、贴边移动和小角度修正不再频繁触发 Pivot，大角度反打仍能稳定进入折返表现。

MM-04

急停滑步：真实速度已下降，Loop / Stop 片段仍持续向前

Stop / Braking 现象

松开输入后真实速度下降较快，但 MM 仍选中 Loop 或长 Stop 片段，Root 位移趋势与制动曲线不一致。

定位

制动参数和停止动画 Root 位移不匹配；停止瞬间 Velocity 特征下降，但 PoseSearch 仍偏向上一段跑步连续性。

处理

建立 Stop 触发窗口，提升速度下降特征权重；调整制动曲线和 Stop 动画选择，必要时用 Stride Warping 收脚。

验证

停止距离可控，脚步相位与胶囊体制动趋势一致，松开输入后不会继续产生明显前向滑移。

MM-05

实例：二期停步排查的通用教训——停步库完好躺在库里，是路由与驱动变量的链断了。MM 里"某类动画不出"优先查路由行序，其次才怀疑动画数据本身。

镜头相对移动方向更新滞后，角色朝向回正慢

Camera-relative Move 现象

玩家快速转镜头并继续移动时，输入方向已经随镜头变化，但角色朝向和移动动画仍沿用旧方向。

定位

镜头方向、期望移动方向、角色朝向和 RootYawOffset 的更新速率不一致，MM 继续选旧朝向片段。

处理

拆分“真实移动方向”和“表现朝向”，缩短高输入期 RootYawOffset 回正；提升未来轨迹方向对选帧的影响。

验证

转镜头后的移动方向能在下一段轨迹采样中被识别，身体朝向保留惯性但不阻塞移动方向更新。

MM-06

移动速度调快后，网络下角色出现拉扯和修正感

Network Feel 现象

为了让起步和折返更灵敏，尝试阶段性提高最大速度，但联机下 Mesh 和胶囊体更容易出现拉扯。

定位

直接改 MaxSpeed 会同时影响真实运动、预测和服务器权威校正；动画层并不一定知道当前处于操作强化期。

处理

将速度增强收敛为可预测的起步 / 折返信号，并通过 SavedMove、配置表和 ABP 参数同步给动画表现层。

验证

起步 / 折返增强在客户端预测与服务器权威结果之间保持一致，减少 Mesh 与胶囊体的修正差异。

MM-07

实例：移动侧配置未进 SavedMove 重放时，客户端预测与服务器结算必然分叉；本项目 StartBoost 曾缺 CanCombineWith 处理，DS 侧告警验证被列入遗留清单——预测一致是竞技化验收的一票否决项。

MM 选帧偶发抽动，同一段移动里步态不稳定

Pose Cost 现象

持续跑动或斜向移动时，角色偶尔切到相位不合适的片段，出现上半身顿一下或脚步突然换节奏。

定位

数据库中存在质量不稳定片段，部分 Start / Loop / Turn 动作标签边界不清；连续性惩罚不足。

处理

清理异常片段，补齐动作元数据；调整 Pose、Velocity、Trajectory、Continuity 的权重关系。

结果

移动中步态连续性提升，少数极端输入下仍可被 Pivot / Stop 明确打断。

MM-08

斜向移动姿态辨识不足，八方向输入差异被削弱

Direction Readability 现象

W+A / W+D 等斜向输入在实机中姿态差异不明显，动作表现更接近前向 Loop 的方向修正结果。

定位

斜向动作覆盖不足，PoseSearch 更偏正向 Loop；Orientation Warping 修正了方向但没有提供足够姿态差异。

处理

补齐斜向移动片段或提高斜向轨迹匹配权重；限制 Warping 过度替代真实方向资产。

验证

斜向输入在肩线、骨盆、脚步方向上有更稳定的姿态差异，八方向移动辨识度提升。

MM-09

实例：与方向性移速衰减（前快侧后慢，MCD 双系数、默认 1.0 逐英雄开启）配套调校——速度模型给了八方向差异之后，姿态层跟不上就会"数值有差、画面没差"。

停下后身体朝向回正慢，下一次起步方向不干净

RootYawOffset 现象

角色停止后仍保留上一段移动产生的身体扭转，下一次起步前 RootYawOffset 先回正再进入移动。

定位

RootYawOffset 半衰期统一配置，不区分高速移动、停止、重新起步等状态。

处理

为 Stop / Start / Pivot 设置不同回正速率；停止阶段允许更快消化旧方向偏移。

验证

停止后的待机朝向稳定，重新起步时不受上一段移动方向残留影响。

MM-10

高速连续变向时脚步贴地性不足

Grounded Feel 现象

快速蛇形走位时，真实轨迹能跟随输入，但脚步接触、重心切换和地面反馈不足。

定位

真实运动响应增强后，动画资源和步幅修正未同步；Stride Warping 过强时会弱化脚步重量。

处理

降低极端步幅修正，增加方向变化时的姿态成本；优先选择重心变化明确的转向片段。

验证

连续走位保持响应速度，同时脚步接触点、骨盆重心和方向切换之间的关系更稳定。

MM-11

低速转向响应不足，角色仍沿用前向 Loop

Low-speed Turn 现象

低速移动或轻推摇杆时，角色方向已发生变化，但动画仍选择前向慢跑 / 行走 Loop，转向姿态不足。

定位

低速区间速度特征过弱，PoseSearch 更容易维持当前 Pose 连续性；低速转向片段覆盖不足。

处理

为低速区间单独提高 Facing / Trajectory 角度权重，补充 Walk Turn / Jog Turn 标签，并限制前向 Loop 在大角度低速下的命中。

验证

低速转向时肩线、骨盆和脚步方向能够随输入变化，小幅走位不再长期保持前向姿态。

MM-12

短按移动导致 Start / Stop 在临界区反复切换

State Hysteresis 现象

玩家短按方向键或轻微修正输入时，Start、Loop、Stop 状态在低速临界区快速切换，表现为步态抖动。

定位

Start 与 Stop 使用同一速度阈值，缺少滞回区间；输入释放后的速度衰减与动画状态退出条件不一致。

处理

拆分 StartEnter、StartExit、StopEnter 阈值，加入最短保持时间和低速输入缓冲，避免同一帧内反复改判。

验证

短按输入可稳定完成一次起停表现，低速临界区不再出现连续状态抖动。

MM-13

起步强化持续时间与输入释放规则不一致

Boost Window 现象

起步强化用于提升初段响应，但在玩家快速松开输入或方向变化时仍可能残留，造成停止距离和下一次选帧异常。

定位

起步强化只按计时结束，未充分绑定输入释放、速度变化和方向反转；ABP 与 CMC 对强化期理解不一致。

处理

明确强化窗口最大持续时间、刷新规则和取消规则；松开输入立即清除强化，折返可刷新但不可叠加。

验证

起步响应保持增强，但停止、短按和连续折返不会继承上一段强化残留。

MM-14

实例：StartBoost 的折返触发网络化在系列内被显式挂为遗留（见六期清单）——触发窗口与输入释放规则没对齐之前，这个爆发不敢放开到网络环境。

Trajectory 预测窗口过长，转向目标被过度平滑

Trajectory Horizon 现象

玩家快速变向时，未来轨迹采样仍保留较多旧方向点，MM 选择偏连续转弯而非即时转向片段。

定位

Trajectory 预测时间跨度过长，历史速度对未来点影响过大；短时间强输入没有足够权重覆盖旧轨迹。

处理

缩短高响应状态下的预测窗口，提高近端采样点权重；在 Pivot / Start 期间优先使用输入方向生成近期轨迹。

验证

快速变向时近期轨迹能更快反映输入，角色不再沿旧方向继续平滑过渡。

MM-15

Trajectory 预测窗口过短，运动趋势识别不足

Trajectory Sampling 现象

缩短预测窗口后，小角度转向较灵敏，但长弧线跑动和持续斜向移动容易选到过于局部的片段。

定位

近端轨迹权重过高导致系统只关注当前输入瞬间，忽略后续运动趋势，Loop 连续性下降。

处理

按状态使用不同采样策略：Start / Pivot 重近端，Loop 重中远端，Stop 结合速度下降趋势。

验证

起步和折返保持响应，持续移动时仍能选择稳定、趋势正确的 Loop 片段。

MM-16

Velocity 与 Acceleration 方向冲突导致选帧不稳定

Feature Conflict 现象

高速反打瞬间，速度方向仍指向旧移动方向，加速度方向已指向新输入，MM 在旧 Loop 与 Pivot 之间摆动。

定位

Velocity 特征和 Acceleration / Input 特征权重没有按状态切换，导致折返初期两个方向互相抵消。

处理

折返窗口内提高 Acceleration / Input 权重，降低旧 Velocity 的连续性保护；普通 Loop 再恢复 Velocity 主导。

验证

反打瞬间能稳定承认新输入方向，持续移动阶段仍保持速度方向一致性。

MM-17

RootYawOffset 累积过量导致身体朝向超调

RootYawOffset Clamp 现象

连续转镜头或反复变向后，RootYawOffset 累积较大，角色身体朝向在回正时出现超调或滞后。

定位

Offset 累积、回正半衰期和最大夹角没有按移动状态区分；高速移动和低速待机共用同一回正策略。

处理

增加不同状态下的 Offset Clamp 和回正速率：高速移动保留惯性，停止 / 起步 / 折返时更快消化旧偏移。

验证

多次变向后身体朝向不会持续残留旧方向，回正过程不产生明显超调。

MM-18

Orientation Warping 过强导致脚尖与膝盖方向异常

Orientation Warping 现象

大角度转向时角色移动方向正确，但下肢骨骼被 Warping 拉到过大角度，脚尖、膝盖和骨盆关系异常。

定位

Warping 承担了本应由转向 / Pivot 动画资产负责的角度差，超过合理修正范围。

处理

为 Warping 设置角度上限和状态条件，大角度转向优先选转向资产，小角度偏差才交给 Orientation Warping。

验证

大角度转向由动画姿态承担，Warping 只做小范围修正，下肢骨骼方向稳定。

MM-19

Stride Warping 补偿过度导致步幅拉伸

Stride Warping 现象

速度调整后，角色脚步能跟上位移，但腿部步幅被拉长，慢速和高速切换时比例不稳定。

定位

真实速度与动画原始速度差距过大，Stride Warping 长期承担速度匹配，掩盖了动画速度档不匹配。

处理

限制 Stride Warping 修正范围，按速度档匹配更接近的 Loop 资产；必要时调整移动速度档与动画库速度标签。

验证

脚步能跟随真实速度，腿部比例不再明显拉伸，速度切换时步幅变化连续。

MM-20

动画库速度档混用，Jog / Run / Sprint 片段边界不清

Database Hygiene 现象

同一移动速度下偶尔选到不同速度档片段，导致步频突然变化，角色移动节奏不一致。

定位

动画库中 Jog、Run、Sprint 的速度标签和实际 Root 速度不完全一致，部分片段跨档命中。

处理

按实际位移速度复核动画标签，清理跨档片段，建立速度档边界和过渡区间。

验证

同一速度区间内步频稳定，进入速度过渡区时变化可预期，不再随机切换节奏。

MM-21

实例：四期 V1 的库治理判据（三选一才留）就是为这一条立的：速度档边界不清的片段宁可清出库，也不留给选帧去赌。

Start / Stop / Pivot 元数据边界不清，状态选帧互相污染

Animation Metadata 现象

起步阶段偶尔选到过早进入 Loop 的片段，停止阶段偶尔选到仍带加速趋势的片段。

定位

动作切片边界、标签和可命中阶段没有明确区分；部分片段同时满足 Start 与 Loop 特征。

处理

重划动画切片边界，补充 StartOnly、LoopOnly、StopOnly、PivotOnly 等命中约束，避免状态互相污染。

验证

起步、循环、停止、折返各阶段选帧更稳定，状态切换不再随机命中相邻阶段片段。

MM-22

连续性成本过高，系统拒绝进入 Start / Pivot / Stop

Continuity Cost 现象

移动很平顺，但起步、折返、急停等关键操作表现不明显，系统长期选择连续 Loop。

定位

Pose 连续性权重和切换惩罚过高，强操作输入无法压过当前动画惯性。

处理

按状态动态调整连续性成本：普通移动保护连续性，强输入窗口降低保护，允许 Start / Pivot / Stop 进入。

验证

普通跑动保持自然，强操作节点能稳定进入对应表现，不再被 Loop 长时间吸附。

MM-23

连续性成本过低，移动中出现片段跳变

Pose Stability 现象

降低连续性保护后，响应提高，但持续移动中更容易跨片段跳转，脚步相位和身体姿态出现突变。

定位

强输入窗口的权重配置泄漏到普通 Loop 状态，导致系统过度追逐瞬时特征。

处理

将响应增强限定在 Start / Pivot / Stop 窗口内，Loop 状态恢复连续性保护和相位约束。

验证

操作节点响应明确，持续移动阶段姿态连续，不再出现无操作原因的片段跳变。

MM-24

移动配置未进入 SavedMove，客户端预测与服务器结果不一致

SavedMove Sync 现象

单机测试移动响应正常，联机测试中起步 / 折返会出现服务器修正，表现为位置回拉或 Mesh 瞬间校正。

定位

部分移动配置、速度强化或状态标记只在本地生效，未完整进入预测移动保存与回放链路。

处理

把起步强化、速度 Buff、配置覆盖、摩擦 / 加速度相关字段纳入 SavedMove 缓存和合并规则。

验证

客户端预测、服务器校验和动画表现使用同一套移动状态，位置修正和表现拉扯减少。

04 / Params 参数框架

实例：与上文网络拉扯条同根因的另一种表象；处理原则一致——移动手感相关的每个配置字段，进 SavedMove 是默认项，不进才需要说明理由。

按问题类型建立可复用的移动参数组

MM 调优需要先判断问题归属，再进入对应参数组：真实位移、PoseSearch 选帧、RootYawOffset、Warping、动画库标签或网络预测。每次修改都通过固定输入序列、录屏对比和调试数据确认变化是否符合目标。

参数组关注问题调优方向风险
起步响应组 按下方向后多久能看到角色承认输入 Start 窗口、初段加速、未来轨迹权重、Start 片段筛选 过强会导致角色弹、动画前摇消失、网络修正感增加
折返 Pivot 组 大角度反打是否稳定进入 Pivot 角度阈值、最小速度、输入保持时间、Pivot 锁定窗口、反向轨迹权重 过强会导致小角度微操也触发大动作
急停组 松开输入后停止距离和滑步量 Braking、Friction、Stop 选帧、脚步相位、Stride Warping 强度 过强会导致停止过硬，动作重量不足
镜头方向组 转镜头后移动方向和身体朝向是否跟手 Control Rotation / Move Direction 关系、RootYawOffset 回正、Facing Cost 过强会牺牲身体惯性，导致朝向变化机械化
选帧稳定组 持续移动中是否抽帧或步态跳变 Pose Cost、Continuity Bias、数据库清理、动作标签边界 过强会让系统不愿意进入 Start / Pivot / Stop
调参顺序建议：先确定真实移动目标，再调 MM 选帧，最后用 ABP 后处理修表现。反过来做，容易用 Warping 或 RootYawOffset 掩盖底层问题，后续越调越乱。 05 / Acceptance 验收方法

固定输入序列验收

MM 移动调优需要固定测试序列，否则某次普通跑动表现正常并不能证明起步、折返、急停和镜头转向都稳定。每次修改后按同一组输入、速度和镜头条件复测，确认是否引入新的边界问题。

基础输入组

示意序列※：前进 1s → 松键急停 → 180° 折返 → 斜后方向打断（各步间隔 0.5s，固定脚本回放）。※ 为示意流程，实际验收以录制脚本为准。

原地起步、短按起步、松开急停、低速转向、高速转向，用来确认移动系统的基础响应和动画衔接。

竞技走位组

ADAD 微操、WASD 斜向、120 度变向、180 度折返、连续蛇形走位，用来判断 Pivot 和方向可读性。

镜头耦合组

移动中快速转镜头、转镜头后立刻反打、镜头与移动方向背离，用来确认身体朝向不会拖输入。

稳定性组

联机测试、低帧率、连续切状态、边界速度，用来排查拉扯、抽帧、滑步和状态误触发。

验收项合格表现不合格表现
起步 输入后角色立即有明确移动意图，动作仍有重心启动 输入已产生但 Start 选帧滞后，胶囊体与 Mesh 初段不同步
折返 大角度反打稳定 Pivot，小角度调整自然转向 180 度绕圈、Pivot 误触发、转身后方向不干净
急停 真实停止和脚步停止趋势一致，停止距离可预期 胶囊体与脚步停止趋势不一致，或制动过硬导致动作重量不足
镜头转向 转镜头后移动方向快速被承认，身体表现有惯性但不拖手 镜头方向已变化，但 MM 仍维持旧方向片段或 RootYawOffset 回正滞后
