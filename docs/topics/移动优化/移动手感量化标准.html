竞技移动手感量化标准

"竞技级手感"这四个字如果不能被测量，就只能被争论。本标准把它拆成五组可测指标：每项给定义、测法、当前值、目标区间：测得出的填实测，测不出的标待测，不达标的直接亮红。以后每一次移动调优，前后各跑一遍记分卡，好坏由数字说话。

定位移动手感的验收锚点：调参手册管"参数是什么"，优化历程管"当时怎么调的"，本标准管"怎么算调好了"。
用法三个场景：调优前后跑记分卡对比；新英雄移动接入的出厂检验；争论手感时先对表，指标都过了还觉得不对，再谈主观。
现状v0.1 记分卡：达标 13 · 待实测 7 · 红灯 4。红灯归并后是三件事：起步误选 Reface、StartBoost 网络同步（含折返爆发）、多人盲测未做，这就是下一期的工作清单。
基线测量环境：PIE 单机 60fps、基准英雄 650 移速档；目标区间标注为 v0.1 建议值，待基线实测后由需求方定稿。

这份标准不打算用数字代替手感判断。它解决的是另一个问题：同一句“更跟手了”，今天、明天、换一个英雄，可能指的不是同一件事。如果没有固定测法，版本之间只能靠当时的手感记忆比较，很难确认是真的变好，还是判断基准变了。

这里把常用的体验词拆成可以复查的代理指标：起步看输入到首帧位移的帧数，急停看松键后的停止距离，折返看速度完成反向所需时间；再用 Rewind Debugger 保留逐帧记录。数字负责发现退化和保证比较口径一致，目标是否舒服仍由实机验收决定。

当前版本是 v0.1 建议稿，多数目标区间还需要基线实测后再拍板。现有记分卡按细项计为 13 项通过、7 项待测、4 项红灯，主要未完成项包括起步误选 Reface、StartBoost 的网络同步和多人盲测。计数口径为：正文五组共 18 项指标，记分卡按拆分后的 24 个计分点统计。在待测项和网络验证完成前，它只能作为验收框架，不能作为“竞技移动已经达标”的证明。

总则

原则与测量环境

项目是 UE5 多人 MOBA 射击，移动层为 GASPALS Motion Matching。本标准约束的是"手感"这一层的验收，不管数值平衡（移速多少归数值，移速兑现得干不干脆归这里）。

三条原则（所有指标由此推导）

一，响应即正义：玩家输入到可感反馈的延迟是第一优先级，表现让位于响应。二，表现不骗人：播出来的动作与角色实际运动一致，动画不许暗示不存在的位移或速度。三，多人一致：本地手感必须在网络下成立，客户端预测与服务器结算一致，单机爽多人回弹等于没做。

测量环境规范

  • 基准场景：PIE 单机，60fps 锁帧（帧数指标按 60fps 换算，1 帧 ≈ 16.7ms）；多人组指标另加局域网双端环境。
  • 基准英雄与移速档：650 移速档（Sprint 区间）为主测档，Run 档抽测；被测英雄需为共享父 ABP 继承链上的标准接入英雄。
  • 输入方式：持续按住方向键测稳态类指标，脉冲输入只用于响应类指标（项目输入是脉冲式，抓帧时容易采到静止帧，测法速查里逐项写明）。
  • 记录手段：Rewind Debugger 录制为准（可逐帧回放），实时 Debug 绘制只做定性初筛。

目标区间的口径

本版目标区间是建议值，每项都附推导理由（人类感知阈值、既有项目实数、本项目已定案的参数），标注为"v0.1 建议"。基线实测跑完一轮后，由需求方逐项拍板定稿，此后目标值变更走修订记录。凭据优先级：本项目实测 > 本项目已定案参数 > 从业经验实数 > 感知阈值文献。

指标五组定义 · 测法 · 当前值 · 目标区间（v0.1 建议）

瞬时响应组（起步 / 急停 / 折返）

竞技移动的胜负手。三个瞬间对应三问：按下去多快动、松开多快停、反打多快掉头。

指标定义与测法当前值目标区间（v0.1 建议）与理由
起步
起步响应按下方向键 → 首帧可感位移（胶囊位移 ≥1cm）的帧数。Rewind 逐帧读胶囊位置待实测（起步冲量已配，体感即时）≤3 帧（50ms）。理由：既有项目实数 50ms 垫步窗口；100ms 是普遍可感知延迟阈值，留一半余量
起步动画完整性起步动画被重选打断的比例（正向直跑起步 20 次）达标 保护窗 1.3s（二期）0 打断。已由 ForceStartTimer 机制保证
起步动画正确性正向直跑起步命中正向起步动画的比例（20 次）未达标 Sprint 档偶发误选反身 180°（Reface，四期未解）100%。当前主红灯，新假设三条待验（见记分卡）
起步 → 巡航交接起步动画结束进入巡航 loop 是否卡过渡（Blend Stack 观察 10 次）达标（Dense 档，四期）；Sparse 未同步修复0 卡滞，两档一致
急停
急停距离650 满速直跑松键 → 胶囊静止的位移（cm）。Rewind 读位置差待实测基线实测后定稿。约束条件先立两条：全英雄同档移速停距一致（公平）；停步动画完整播出不被裁（表现）
停步动画命中松键出停步动画（而非巡航渐停）的比例达标（二期恢复停步链路）100%
折返
折返耗时满速直跑反打方向 → 速度分量反向的耗时（ms），及折返动画命中率待实测（链路已通，二期）基线实测后定稿；动画命中 100%。注意 Sprint 折返库仅 4 条的覆盖限制
折返爆发折返瞬间速度曲线是否有确认感（对比无 boost 曲线）未达标 StartBoost 折返触发未启用（网络同步缺口，六期遗留）启用后单独验收，多人一致性组联测

巡航与转向组

指标定义与测法当前值目标区间（v0.1 建议）与理由
巡航稳定性满速直跑 10 秒内 Blend Stack 跳姿势次数（loop 自然循环不计）达标（四期限采样后"现在稳了"）；量化基线待补测0 次。巡航期任何重选都是波浪的种子
躯干稳定性直跑时躯干横向摆动幅度（目测定性 + 必要时录屏逐帧）达标（二期 FootPlacement 战役清零）无可感晃动。回归项：改 FootPlacement 链路或补烘 footspeed 曲线后必须复测
低速滑步低速走位（<200cm/s，补刀卡位场景）脚部与地面的相对滑动，脚速曲线与位移速度比对待实测（StrideWarping 已配，MOBA-Lite 裁 Walk 后的既定代偿）拉远视角无可感滑步。这是裁库决策的验收承诺，必须盯
变速跟随移速 buff 生效瞬间（如 650→595 或反向）动画档位切换正确性与播放速率平滑达标 Gait 移速属性驱动 + 迟滞 560/480（四期）；PlayRate (0.85,1.10) 吸收失配无临界抖动；playrate 不贴钳制边缘（贴边 = 数据失配预警，见手册 05）
转向表现strafe 模式下快转镜头：下肢对移动向、上身对镜头的扭身分配是否自然，无拧麻花达标（三期 OrientationWarping 定案：spine 分摊 0.5、135 阈值）无反向过旋、无麻花；大角度反身有转身动作（保留 Reface 的设计承诺）
方向性移速前 / 侧 / 后移速比例与过渡连续性（斜向无突变）达标 前 1.0 / 侧 0.9 / 后 0.8，余弦缓入连续插值（六期）比例为数值层决策；手感验收只管连续性：任意方向摇杆画圆，速度无跳变

技能期间的移动（与技能系统的接缝）

互斥语义（谁挡谁、谁断谁）已由《技能释放关系系统》管辖，本节不重讲机制，只收移动侧的手感指标：位移的权限、惯性与恢复。

指标定义与测法当前值目标区间（v0.1 建议）与理由
位移收尾惯性位移技能（冲锋类）结束瞬间的速度曲线：是否继承残余速度滑行收束，还是原地急停达标（Steel 冲撞收尾继承残余速度前滑，MaintainLastRootMotionVelocity 先例）位移结束不许急停砸地，残速平滑衰减。这是"位移手感原则"，新位移技能出厂检验项
蒙太奇期间移动可移动的全身攻击：收招段腿部交还 locomotion 的时点与平滑度（Rewind 看 Slot 权重与腿部姿势来源）达标 Kallari 案：0.543s 起交接、0.1s 混合、结束权重归零无卡 Pose（六期终案）交接点 = 动画 FullBody 曲线归零点 ±3 帧；无卡 Pose、无腰胯跳变。新的可移动攻击照此验收
位移期间输入权RMS 位移期间移动输入被压制与恢复的帧数（位移结束 → 输入重新生效）待实测（规则已有，恢复帧数未量化）恢复 ≤2 帧。位移结束后输入被吞是投诉高发区
3C 动作与技能互斥翻越/攀爬期间技能封锁与解锁时点的一致性（关系表 DT 行为抽测）达标（关系表 DT 化，编辑即生效；翻越挡技能已配）封锁解锁与动画窗口一致，无"动作结束了还锁着"的死区。配置改动后跑抽测
受控恢复眩晕/定身解除瞬间移动恢复的帧数与姿势衔接（不 T-pose、不滑行）达标（Lissandra 案修复后：解冻瞬间蒙太奇停止、混回 locomotion）解除 ≤2 帧恢复输入；定格类蒙太奇必须配 Hold 循环段防孤儿（Lissandra 案沉淀的红线）

接缝类指标的通用红线：任何"技能占用移动"的状态，都必须有确定的归还路径（正常结束、被打断、网络回滚三条路都要还）。Lissandra 案和卡 Pose 案的共同根因都是"占用有路、归还没路"。

多人一致性组

三原则的第三条。单机指标全过之后，同一批指标要在网络下重跑，另加下面四项网络专属。

指标定义与测法当前值目标区间（v0.1 建议）与理由
预测一致性本地预测与服务器结算的位置修正幅度（Rewind 看回滚事件与修正距离），重点场景：起步、折返、方向切换、buff 变速瞬间部分达标 方向衰减取 Acceleration（重放数据）预测安全（六期）；StartBoost 未过（瞬时帧被移动合并吃掉 → 回弹，缺 CanCombineWith）常规移动 0 可感回弹；瞬时状态（起步/折返 boost）启用前必须过合并比较补全
远端表现simulated proxy 视角下他人移动的动画正确性（起步/急停/折返是否与本人视角一致）待实测（StartBoostTimer 对远端的复制待核，交接遗留项）远端与本端动作语义一致，允许细节降级不允许语义错位（本端折返远端滑移 = 不合格）
DS 干净度专用服务器上移动/动画链路的日志与开销（warning 清零、无每帧高频查找）待验证（调试 CVar 支路已断，等 DS 侧同步确认）0 warning；编辑器专属调试链路不进 DS
多人盲测局内多人对局的手感盲测（参与者不知改动内容，前后版本对比打分）未做标准定稿前至少一轮。单人 PIE 的"跟手"在 100ms 网络下可能完全是另一回事，这是全表最诚实的一行

测量方法速查

工具细节在《调参手册》第 10 节工具箱，这里只列"指标 → 用什么测"的对应：

指标类型工具与手法注意
帧数 / 耗时类（起步响应、折返耗时、恢复帧数）Rewind Debugger 录制 → 逐帧读胶囊位置与速度；输入时刻以增强输入事件帧为零点项目输入是脉冲式，测响应必须录制回放，实时抓帧会采到静止帧
距离类（急停距离）Rewind 读松键帧与静止帧的胶囊位置差；同英雄测 5 次取中位数坡面与平地分开测，先只立平地基线
动画命中类（起步/停步/折返命中、误选率）Blend Stack 调试面板 + Rewind 混合权重轨道，蓝色高亮为主选命中率类每项 ≥20 次采样；playrate 贴钳制边缘要单独记录（数据失配预警）
滑步类脚部骨骼速度曲线与胶囊位移速度比对；拉远至实战镜头距离做可感性终判可感性以实战镜头为准，别在特写镜头下追求绝对零滑
网络类（回弹、远端表现）局域网双端 + 模拟延迟（100ms 起）；Rewind 双端各录一份对比；p.NetShowCorrections 看修正先关 StartBoost 类瞬时逻辑测基线，再开测增量
逐帧移动记录（待建）参照镜头侧 camera_trace.py 的做法出一个 movement_trace 脚本：逐帧记录 位置/速度/加速度/当前动画/playrate 落 CSV建成后上面多数手测项可以脚本化，回归成本降一个量级

记分卡 v0.1（2026-07-13）
