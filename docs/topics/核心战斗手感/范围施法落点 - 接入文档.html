概述

一、这是什么

一套统一的落点决策系统。技能要把东西放到地面上时（召唤一个单位、铺一片 AOE、传送到某点），落点不再各算各的，而是统一交给 UPlacementResolverLibrary 这个纯函数算法库决定。

三个关键特性，接入前先知道：

  1. 三端同源：客户端预览、客户端释放、服务端反作弊复算，跑的是同一份算法。你预览看到的落点 = 实际落点，一帧不差。
  2. Emergent LOS（撞哪算哪）：落点由相机射线的几何自然决定——射线被墙挡住就留这侧，越过墙顶就到那侧。无需配任何"视线遮挡"参数。
  3. 永远有合法落点：任何瞄准都返回一个合理落点，没有"无法施放"的失败态（Deadlock 派设计）。

二、接入步骤（策划三步，不改代码）

技能模块（SkillModule）侧已经接好了，策划只需要在技能的 Preview DataAsset 上配置：

  1. 打开技能的 Preview 资产（SummonPreviewDataAsset 或 SkillAreaPreviewDataAsset）。
  2. 填 PlacementProfile：先选 Mode，再调 MaxRange 等参数（见第三、四节）。
  3. 存盘、进游戏验证：按住技能键看预览虚影，落点即为最终落点。

老技能资产无需手动改——加载时 PostLoad 会自动把旧字段迁移到 PlacementProfile（见第八节）。

三、PlacementProfile 字段参考

字段 含义 参考值 说明
Mode 落点模式 Deployable 见第四节。决定走哪套逻辑
MaxRange 最大水平射程（cm） 2000 超出后沿瞄准方向夹回，再向下找地面
SurfaceChannel 落点检测的碰撞通道 ECC_SummonPreview 决定"哪些物体算地面/墙"。一般不动
UpwardTraceHeight 向上检测高度（cm） 250 夹回/兜底时向上找地面的起点高度
DownwardTraceHeight 向下检测距离（cm） 2000 向下找地面的最大距离，要足够大能打到地面
MinSurfaceNormalDotUp "算地面"的法线阈值 0.1 面的朝上程度 ≥ 此值算可站立地面，否则算墙（走墙面逻辑）

四、Mode 怎么选

Mode 用途 行为
Deployable 召唤物、部署物（最常用） 落点接地。走完整落点逻辑（含墙面处理）
GroundAOE 地面范围技能 同上，落点接地
None 不接入本系统 保留旧的"相机锚定"预览行为。零回归——老 AOE 不想改的就留 None
ProjectileLanding （V2 预留） 暂勿使用，V1 会按 Deployable 处理并报 ensure 提醒

选择建议：召唤/部署类一律 Deployable；只想要个跟随相机的范围圈、不需要接地的，用 None。

五、支持的技能模块（消费方）

以下 SkillModule 已接入本系统，配好 Profile 即生效：

  • SkillModuleSpawnMobaUnit —— 召唤单位（SpawnType = BasedOnPreviewData）
  • SkillModuleAddSkillAreaWithBuffs —— 范围 + Buff
  • SkillModuleTeleportToTarget —— 位移（BasedOnSummonPreview / BasedOnSkillAreaPreview 模式，传送到落点）

注：TeleportToTarget 的 RelativeToSelf / BasedOnArmedTarget 模式走各自逻辑，不经本系统。

六、落点行为（配参数前先理解）

相机射线打到哪，决定落点：

  • 打到平地（法线朝上）→ 直接用命中点。涵盖"从高处俯瞄平台顶面"。
  • 打到墙（法线偏水平）→ 墙面三段式：根据瞄准高低，自动判断玩家要"上墙顶/平台"还是"留在墙根地面"。
  • 打空（瞄天 / 地图外）→ 沿瞄准方向兜底到 MaxRange 处找地面。
  • 最后统一做 MaxRange 水平夹回。

墙面三段式借鉴了角色攀爬系统（CanMantle）的"探测最近可放置面"思路——瞄墙上部命中点离墙顶近就上去，瞄墙下部离地面近就留下。

七、复杂立体场景处理

平地谁都能算对，立体地形才是重点。本系统正确处理：

场景 玩家意图 落点
悬浮桥 / 高架 瞄桥侧壁上部 上桥面
瞄桥侧壁下部 桥下地面
站桥上俯瞄桥面 桥面
夹层 / 二楼 瞄外墙上部 上二楼
瞄外墙下部 留当前层地面
矮墙 / 栏杆 视线越过矮墙 落对面地面
视线被墙挡 落墙这一侧
台阶 / 斜坡 瞄坡面/台阶 贴合落点，不悬空不嵌入
顶棚 / 洞穴 仰瞄顶棚 落脚下地面（不卡天花板）
地图边界 瞄地图外 前方射程内找合法地面

一句话：凡是"看得到、够得着"的地方落点就符合直觉；立体地形的上下分层，系统能读懂你瞄的是哪一层。

八、老资产迁移（零回归）

SummonPreviewDataAsset 在 PostLoad 自动迁移：旧字段（SummonPreviewRange / UpwardTraceHeight / DownwardTraceHeight 等）会自动复制到 PlacementProfile，且 Mode 设为 Deployable。

  • 老召唤技能无需手改，加载即走新逻辑。
  • 如需保留旧"相机锚定"行为，手动把 Mode 改回 None。

九、反作弊（程序侧，策划了解即可）

服务端用同一算法复算落点，与客户端发来的坐标比对：

  • 偏差 ≤ 200cm → 采用客户端坐标（手感无突变）。
  • 偏差 > 200cm → 采用服务端坐标 + 记审计日志（挡 cheat）。

策划无需配置，开箱即用。

十、调试

控制台开关，定位落点问题用：

p.PlacementDebug 1

开启后日志（[PlacementDBG]）会打印：相机射线命中点 / 法线 / 走的分支（平地 / 墙 / 落空）/ 上下探测结果 / 最终决策。

该日志 #if !UE_BUILD_SHIPPING + CVar 默认关，不影响打包，平时零开销。

十一、注意事项 / FAQ

  • 预览和实际不一致？ 不应该发生（三端同源）。若出现，开 p.PlacementDebug 看日志，多半是 SurfaceChannel 配置或 mesh 碰撞设置问题。
  • 召唤物嵌墙 / 太靠墙？ 落点 XY 贴墙根是预期；若召唤物 mesh 较粗，体积裁切（Footprint）属 V2 范畴，与落点决策无关。
  • DownwardTraceHeight 设多大？ 要足够从最高可能命中点向下打到地面。默认 2000cm 适用绝大多数场景。
  • 想让某技能不接入？ Mode = None。
