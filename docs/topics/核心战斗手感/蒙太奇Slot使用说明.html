概述

0. 一句话原则

Slot 不是越多越好，它决定的是"这段动画怎么混进当前 base pose"。选错了，再好的资源也是抽搐的。

1. 当前可用的 Slot 一览

所有 Slot 都在 Skeleton 的 Anim Slot Manager 里，统一挂在 DefaultGroup 下。

Slot 名 用途 半身？ 是否会被下半身姿势"带歪" 在 ABP 里走哪种节点
DefaultSlot 整体替换：mantle、大招、过场、复活 全身 — 直接覆盖（不走 LBPB）
UpperBody 硬上半身：持盾、瞄准、定身释放等强语义动作 上半身（spine_01 以上） ❌ 完全不传导 LBPB + Mesh-Space Rotation Blend，BlendMask = BP_UpperBody_Hard
UpperBodySoft 软上半身：要求和跑动 / 转身协调的能力动作 上半身（spine_02 以上，肩→手） ✅ pelvis / spine_01 跟随 base LBPB + Mesh-Space Rotation Blend，BlendMask = BP_UpperBody_Soft
UpperBodyAdditive 叠加：受击反馈、呼吸、轻微反馈 Mesh-Space Additive ❌ 不替换，纯 add ApplyMeshSpaceAdditive（不走 LBPB）
PreAO_UpperBody GASPALS overlay 系统专用（持枪叠加等） 半身 — overlay 内部接线，业务侧不要用
Transition MM / 状态机内部过渡 — — 状态机内部用，业务侧不要用
Face / Fingers 局部（脸 / 手指） 极小骨子集 — 局部 LBPB（按需启用）

2. 怎么选：决策树

要播的这段动画——

① 是否要替换整个身体？（mantle、大招、过场、死亡复活）
   ├─ 是 → DefaultSlot
   └─ 否 → ②

② 这段动画是「叠加」性质吗？（受击抖动、呼吸、轻微反馈、不想替换 base）
   ├─ 是 → UpperBodyAdditive
   │       ⚠ 前提：动画文件本身 AdditiveAnimType 必须是 mesh-space additive
   └─ 否 → ③

③ 释放过程中，角色是否需要继续跑 / 继续转向？
   ├─ 不需要（定身瞄准、起手蓄力） → UpperBody （硬）
   └─ 需要继续跑或转向            → UpperBodySoft （软）

3. 三种主上半身 Slot 的差异（最常用的就这三个）

维度 UpperBody UpperBodySoft UpperBodyAdditive
替换 or 叠加 替换上半身 替换上半身 叠加到 base 上
pelvis / 髋是否被下身带 ❌ 不带 ✅ 跟随下身 ✅ 完全由下身决定
适合的动画语义 持盾、瞄准、定身起手 普攻 release、追击释放 受击、抖动、反馈
动画文件要求 普通动画 普通动画 必须是 mesh-space additive，base 用对应角色 Idle_Combat
用错的代价 跑动时上身僵 起手不够稳 上身抽搐 / 飘移

4. 实战案例（来自项目踩坑）

案例 A：Steel 普攻"飘"

  • 症状：跑动 / 转身时 Steel 普攻上身不跟，看着像悬浮。
  • 当时配置：普攻 montage slot = UpperBody（硬）。
  • 真因：普攻动画是要边跑边出招的，硬 mask 把 spine_01 锁死了，导致上身和下身脱节。
  • 修复：动画文件改回 AAT_NONE → montage slot 改成 UpperBodyAdditive，并把 base 设成 UEFN_rtgSteel_Idle_Combat。
  • 教训：普攻 release 类不要塞 UpperBody。要么走 UpperBodySoft（普通动画），要么走 UpperBodyAdditive（additive 动画）。

案例 B：把 additive 类型动画塞进 UpperBody

  • 症状：上身整个抽搐 / 扭曲。
  • 真因：UpperBody 走的是 替换型 LBPB，base pose 被直接覆盖成 additive 的 delta。Additive 是"差量"，本来是要 add 在 base 上的，被当成绝对姿势放进去就是抽搐。
  • 修复：检查动画的 AdditiveAnimType：
  • AAT_NONE → 走 UpperBody 或 UpperBodySoft
  • AAT_LocalSpaceBase / AAT_MeshSpaceBase → 只能走 UpperBodyAdditive

5. 注意事项

5.1 选 Slot 之前先看动画的 AdditiveAnimType

打开动画资源 → Additive Settings → Additive Anim Type：

AdditiveAnimType 能进哪些 Slot
AAT_NONE（普通动画） DefaultSlot / UpperBody / UpperBodySoft
AAT_LocalSpaceBase 或 AAT_MeshSpaceBase（叠加动画） 只能 UpperBodyAdditive

5.2 Additive 动画的 base 要选对

UpperBodyAdditive 里的动画一定要用对 base：

  • 战斗态动作 → base 选角色对应的 XXX_Idle_Combat
  • 非战斗 → base 选 XXX_Idle

base 不匹配的话，叠加出来的姿势会偏。

5.3 不要碰这些 Slot

  • PreAO_UpperBody、Transition：是 overlay 系统 / 状态机内部用的，业务侧 montage 别选它们。
  • Face / Fingers：表情、手指系统专用。

5.4 数值不对就先回到 Slot 这一步排查

技能视觉异常的第一排查顺序：

  1. 动画的 AdditiveAnimType 对吗？
  2. Slot 选对了吗？
  3. base 选对了吗（UpperBodyAdditive 才需要看）？
  4. 才轮到调 BlendIn / BlendOut 时长。

6. 改 Slot 之后

如果你改了任何 montage 的 Slot，做完一个简单自测：

  1. PIE，角色站定 → 触发这段动画 → 上身正常
  2. PIE，按住前进让角色跑动 → 触发这段动画 → 上身不该抽搐 / 飘
  3. PIE，跑动中快速左右转向 → 触发这段动画 → 上身和下身该协调

三种情况都正常才算过。
