Combat Tech Design Requirement

TPS 实体弹道发射点与瞄准意图统一规范

面向 Twinblast 普攻与后续 TPS 实体弹道技能，明确“相机负责瞄准意图，枪口负责真实发射”的需求边界、配置方式、表现要求和验收标准。

1. 背景

当前 Twinblast 普攻的即时子弹在逻辑上更接近从相机射线发出，枪口 socket 主要用于视觉偏移。这样能保证准星命中感，但会带来 TPS 中常见的公平性问题：角色或武器仍在掩体后，子弹却从镜头线命中目标。

本需求的核心结论：
  • 玩家的瞄准意图由相机准星决定。
  • 实体子弹的真实出生点由武器口或逻辑枪口决定。
  • 子弹方向由枪口指向相机解析出的 AimPoint。
  • 需要支持旧逻辑兼容，不能一次性改变全项目所有投射物手感。
不要把本需求理解成“所有技能都从枪口发射”。 这是一个可配置的发射策略需求，优先服务 Twinblast 普攻和同类 TPS 实体子弹。

2. 范围

本文档只定义实体子弹发射起点与准星意图的需求，不替代动画、武器绑定、AO、IK、命中伤害和网络预测文档。

范围项 是否纳入 说明
Twinblast 普攻即时子弹 P0 首个接入目标。当前资产已有 IPD_TwinblastAttack 与 hand_l / hand_r 多枪口配置基础。
其他英雄普攻即时子弹 P1 待 Twinblast 验证后按技能类型逐步接入，不做全量自动迁移。
LifeTime Projectile P1 逻辑相似，但生命周期、预测和碰撞成本更高，建议第二阶段接入。
Beam / Laser P2 需要单独定义持续射线、遮挡更新、端点吸附和表现拉线规则。
AO、右手 IK、开火 Overlay 不纳入 属于身体资源和动画表现并行问题。本文只要求枪口位置可被正确读取和验收。
伤害数值、射速、弹道速度平衡 不纳入 发射点改变可能影响手感，但不在本文内调整数值。

3. 当前工程现状

当前工程已经有即时子弹系统和视觉枪口配置，但逻辑发射点与视觉发射点不是同一概念。

模块 / 资产 当前职责 问题
FSkillModuleContinuousFire 技能激活后按射击间隔启动连续即时子弹。 只负责启动连续开火，不决定每发子弹的真实起点。
UInstantProjectileSystem::SpawnAndLaunchInstantProjectileFromContinuousFire 每次开火时从控制器获取相机视角，计算 StartLocation 和 LaunchVelocity。 当前真实 StartLocation 来自相机视点到角色位置的投影，不是武器口。
USkillProjectileDataBase::VisualStartSocket 配置表现从哪个骨骼或 socket 出现。 当前主要影响视觉偏移，不等于逻辑发射点。
IPD_TwinblastAttack 配置 Twinblast 普攻即时子弹。 已开启多枪口表现，数组为 hand_l、hand_r，但逻辑仍不是从这些点发射。
IPD_TwinblastAttack_R 右手版本即时子弹配置。 配置了 hand_r，仍只作为表现枪口。
当前项目已经具备“从 socket 找表现枪口”的基础，缺的是“发射策略配置”和“逻辑发射点使用 socket”的代码路径。

4. 使用场景

需求先从玩家感知最强、最容易出问题的场景出发。

场景 需求 验收关注点
常规站立开火 准星决定目标方向，子弹从当前发射手的枪口或逻辑枪口出生。 调试线显示 MuzzleStart 与实际子弹起点一致。
左右手交替开火 Twinblast 每发根据开火序号选择左手或右手枪口。 第 1 发、第 2 发分别从预期 socket 出生；视觉枪口和逻辑起点一致。
肩射贴墙 如果准星能看到敌人，但枪口被墙挡住，子弹不得从相机穿墙命中。 子弹应命中遮挡物，或 UI 给出枪口遮挡提示。
近距离贴脸 相机命中点过近时，需要避免枪口方向大角度横甩。 近距离目标命中稳定，不出现明显反向发射或横飞。
俯仰高低差 枪口方向应收敛到相机解析出的目标点，不能只用枪口 forward。 上坡、下坡、空中目标下准星与命中趋势一致。
非 Twinblast 旧技能 默认不改变旧技能逻辑。 未接入新策略的投射物表现和命中手感保持不变。

5. 核心规则

系统应采用“相机解析意图，枪口兑现物理”的规则。规则顺序必须稳定，否则调试和验收都会困难。

规则 说明 不允许
Camera 只表达意图 从准星中心发出 CameraRay，解析 AimPoint。 不允许直接把相机位置当作实体子弹出生点。
Muzzle 表达物理来源 实体子弹从 LogicalMuzzle 或指定 socket 出生。 不允许逻辑从相机出，视觉再从枪口补线伪装。
方向由枪口指向意图点 LaunchDirection = Normalize(AimPoint - MuzzleStart)。 不允许只用枪口 forward，导致准星和命中趋势分叉。
近距离需要保护 AimPoint 过近时使用最小收敛距离或远点回退。 不允许近距离下子弹大幅横飞、反向飞或穿过自己身体。
遮挡必须可信 枪口到目标方向被近处障碍挡住时，应表现为打墙或无法射击提示。 不允许角色躲在墙后通过镜头线命中敌人。

6. 配置需求

策划需要能按投射物配置发射起点策略。默认值必须兼容现有项目。

配置项 推荐值 需求说明
LaunchOriginPolicy CameraProjected 默认保持当前逻辑。可选值建议包含 CameraProjected、VisualStartSocket、CustomSocket。
VisualStartSocket 按资产配置 单枪口模式下作为表现起点；当策略为 VisualStartSocket 时，也可作为逻辑起点。
MultipleVisualStartSocketArray 按资产配置 多枪口模式下根据开火序号选择当前枪口。Twinblast 当前为 hand_l、hand_r。
MinimumAimDistance 建议 800-1200 当相机命中点过近时，用于把 AimPoint 推到合理距离，避免近距离方向剧烈偏转。
bCheckMuzzleBlocked 建议 P1 开启 控制是否检测枪口近距离遮挡。P0 可先用真实枪口发射后的碰撞结果体现遮挡。
MuzzleBlockedFeedback 后续 UI 配置 枪口遮挡时准星、音效或提示表现。该项不阻塞 P0。
Twinblast 建议： 先把 IPD_TwinblastAttack 的新策略配置为 VisualStartSocket。如果后续武器资产存在更准确的 muzzle_l / muzzle_r，应优先使用真实枪口 socket 替代 hand_l / hand_r。

7. 表现与交互要求

发射点改变后，玩家会更明显地感知到枪口、准星、弹道、遮挡之间的关系。表现层需要配合解释系统规则。

表现项 需求 验收方式
枪口火光 火光和实体子弹出生点一致，左右手交替时不能串手。 慢速播放开火，确认第 N 发对应第 N 个 socket。
弹道 / Beam 起点 弹道表现从当前枪口开始，不再从准星中心或相机附近出现。 开启 debug 线，对比 VisualStartLocation 与 StartLocation。
准星命中感 中远距离下仍应表现为打准星指向位置。 站立射击固定靶，中心准星命中趋势稳定。
枪口遮挡提示 P1 需要准星或 UI 提示当前枪口被遮挡。 贴墙瞄准敌人时，玩家能理解为什么子弹打墙。
手部动画 本文不要求修正 AO/IK，但要求 socket 位置可被稳定读取。 开火期间 socket 不应跳到角色内部或异常位置。

8. 兼容与迁移

发射起点会直接影响命中手感和 PVP 公平性，因此必须小步迁移。

阶段 迁移对象 要求
阶段 1 Twinblast 普攻 只接入 IPD_TwinblastAttack，并保留回退到旧逻辑的配置能力。
阶段 2 Twinblast FastFire / 右手版本 确认普通开火稳定后，再接入同角色其他即时子弹。
阶段 3 其他英雄普攻 按角色逐个验收，不做批量默认切换。
阶段 4 特殊投射物与 Beam 根据技能语义单独配置，避免普通子弹规则误伤技能表现。
旧资产默认继续使用 CameraProjected。只有明确配置为 VisualStartSocket 的投射物才改变真实发射点。

9. 优先级与验收

每条需求必须能被策划、程序或 QA 直接检查。

优先级 需求 验收标准
P0 支持按投射物配置发射起点策略。 旧资产默认不变；Twinblast 可单独切到枪口发射。
P0 Twinblast 普攻从当前左右手枪口或逻辑枪口出生。 开启 debug 后，实体子弹 StartLocation 与当前发射 socket 位置一致。
P0 准星仍负责瞄准意图。 中远距离打靶时，子弹方向收敛到准星解析出的 AimPoint。
P0 旧连续开火逻辑可回退。 把 Twinblast 策略改回 CameraProjected 后，表现和当前版本一致。
P1 枪口遮挡检测与提示。 贴墙瞄准时不发生相机穿墙命中，UI 或准星能提示遮挡。
P1 最小瞄准距离保护。 贴脸或近距离地面命中点不会导致子弹大角度横飞。
P2 扩展到 Beam / Laser / LifeTime Projectile。 每类投射物有独立验收，不复用即时子弹结论硬套。

10. 程序备注

本节给程序实现参考，不作为策划正文需求的替代。

建议新增发射起点策略

enum class EProjectileLaunchOriginPolicy : uint8
{
    CameraProjected,     // 当前逻辑，默认兼容旧资产
    VisualStartSocket,  // 使用 VisualStartSocket / MultipleVisualStartSocketArray 作为逻辑起点
    CustomSocket        // 后续扩展：单独配置逻辑枪口 socket
};

建议运行流程

CameraRay = GetPlayerViewPointForGameplay()
AimPoint = ResolveCameraAimPoint(CameraRay, MaxRange, MinimumAimDistance)

if LaunchOriginPolicy == VisualStartSocket:
    MuzzleStart = ResolveCurrentVisualStartSocketLocation(CustomFireIndex)
else:
    MuzzleStart = CurrentCameraProjectedStartLocation

LaunchDirection = Normalize(AimPoint - MuzzleStart)
StartLocation = MuzzleStart
LaunchVelocity = LaunchDirection * LaunchSpeed

当前可能涉及的代码点

文件 备注
SkillProjectileDataAsset.h 适合新增策略配置。默认值必须为当前逻辑。
InstantProjectileSystem.cpp SpawnAndLaunchInstantProjectileFromContinuousFire 是 Twinblast 连续开火每发子弹起点的关键入口。
InstantProjectileSystem.h 可新增 socket 解析和枪口位置解析辅助函数。
IPD_TwinblastAttack 接入新策略后的首个配置资产。
实现风险： 如果真实枪口 socket 配在武器 mesh 而不是角色 mesh 上，当前只查角色 mesh 的逻辑会找不到准确枪口。需要确认 Twinblast 的真实枪口 socket 所在组件。

11. 待确认

以下问题会影响 P0 的最终表现，但不影响先建立可配置策略。

问题 建议决策 影响
Twinblast 是否已有真实 muzzle_l / muzzle_r socket？ 优先使用真实枪口；没有则 P0 先用 hand_l / hand_r 验证流程。 影响子弹出生点精度和视觉可信度。
socket 在角色 mesh 还是武器 mesh？ 若在武器 mesh，需要提供统一武器组件查询入口。 影响程序读取枪口位置的方式。
贴墙时是允许开火打墙，还是直接禁止开火？ 建议 P0 允许开火并打墙，P1 增加准星遮挡提示。 影响玩家反馈和输入响应。
双枪真实发射是否会导致左右边缘命中不同？ 先接受真实枪口差异；如果手感过抖，再评估逻辑中点枪口。 影响 PVP 边缘命中手感。
