Planning Requirement / Combat 3C

Skill - 范围施法落点需求

面向第三人称 MOBA / Hero Shooter 的地面范围、召唤、陷阱和投射落点技能。目标是让预览、释放和服务器判定使用同一套落点规则：玩家给出意图，系统只做同语义修正，不替玩家选另一个位置。

需求方：战斗 / 3C 策划 实现方：战斗程序 / UI / 网络 适用：Ground AOE / Deployable / Projectile Landing 版本：V1 精简需求稿

1. 背景与目标

范围施法落点如果每个技能各写一套规则，最容易出现三类问题：预览能放但释放失败、释放位置和预览位置不一致、边缘或高低差场景被吸到玩家没想选的位置。这里统一落点决议口径，让策划配置、客户端预览、服务器复检和 QA 验收说同一种语言。

一句话规则

系统可以把玩家的原始落点修正到“同一语义位置”的合法点；不能把楼上修到楼下、墙前修到墙后、目标地面修成另一个平台，也不能在失败时悄悄消耗技能。

需要解决

  • 地面范围圈、召唤物、陷阱、投射落点使用统一判定流程。
  • 预览表现和正式生效位置一致，修正后的最终落点必须可见。
  • 非法原因可枚举，失败时能给 UI、网络和 QA 明确结果。

交付结果

  • PlacementProfile：技能落点配置。
  • PlacementResolver：客户端和服务端共用的决议函数。
  • ResolvePolicy 与 FailureReason：修正规则和失败原因。

边缘部署

同一桥面内可以贴边修正；桥下是另一个语义位置，不能自动掉层。

抛物线落点

Reticle、范围圈和服务器命中点必须是一回事，不能只让轨迹看起来对。

预览承诺

玩家松手前看到的状态要可信：合法就释放，修正就显示修正点，非法就明确失败。

2. 本版范围

范围 本版要求 说明
地面范围技能 必须接入统一落点流程。 圆形、扇形、矩形等范围先得到 FinalPlacement，再进入伤害或区域系统。
召唤 / 陷阱 / 放置物 必须校验占地、阻挡、导航和高度。 不能只判断中心点合法，必须判断实际 Footprint。
投射落点 / 抛物线终点 预览终点和服务器终点需要共享落点口径。 飞行过程的命中仍由投射物系统负责，本稿只定义“落到哪里”。
墙面 / 天花板 / 任意 3D 放置 不进入 V1。 需要单独的表面朝向、附着、遮挡和动画规则，避免拖慢地面技能落地。
自动替玩家选目标 不进入本系统。 锁敌、辅助瞄准、目标选择归目标选择系统；落点系统只处理空间位置是否合法。
V1 先把“地面落点可信”做稳。特殊技能可以先不接入，但接入后必须遵守同一套预览、释放和服务器复检口径。

3. 术语

术语 含义 验收口径
AimRay 玩家当前瞄准方向，通常来自镜头或准星。 客户端预览和服务端复检应能得到同类输入。
IntentPoint 由瞄准射线或最大距离投影得到的原始意图点。 用于表达玩家想点哪里，不一定合法。
SupportSurface 承载落点的地面或平台。 同一支撑面内可以修正，跨支撑面默认失败。
Footprint 技能实际占地形状，可能是点、圆、盒、胶囊或自定义形状。 放置物和范围技能必须检查完整占地。
PlacementProfile 技能落点配置，包括形状、距离、表面、修正策略和失败反馈。 策划配置入口，不在技能蓝图里散写找点逻辑。
FinalPlacement 最终用于释放、生成、伤害范围、VFX 和网络同步的位置。 正式表现必须以它为准。
FailureReason 落点失败原因。 UI、日志、QA 用同一份枚举，不用猜失败原因。

4. 落点流程

所有接入技能按同一条流水线求值。客户端预览可以高频调用，服务器释放时必须用同一套配置重新求值。

步骤 输入 输出 要求
1. 构建上下文 施法者、瞄准、技能配置、当前移动状态 PlacementInput 预览和释放使用同一类输入结构。
2. 求原始意图点 AimRay、最大距离、准星命中 IntentPoint 只表达玩家想点哪里，不在这一步做吸附。
3. 找候选面 IntentPoint 附近的地形 / Nav / 碰撞 SupportSurface 默认优先同一支撑面，不跨平台找点。
4. 校验占地 候选面、技能形状、阻挡信息 合法 / 非法原因 范围和放置物都按完整 Footprint 判断。
5. 执行修正策略 非法原因、容差配置、候选点 FinalPlacement 或 FailureReason 修正结果必须能反馈给玩家。

5. 合法性规则

规则 默认要求 失败原因
距离 落点必须在技能最大距离内；超出距离时可沿同一方向夹取到最大距离。 OutOfRange
支撑面 落点必须在允许的地面类型上，默认不允许墙面、天花板、空中点。 InvalidSurface
高度 同一支撑面内允许小高度差；跨楼层、跨平台、穿过断崖默认失败。 HeightMismatch
占地 Footprint 内不能与不可穿越物、角色胶囊或禁止区域冲突。 FootprintBlocked
导航 需要落地实体的技能必须落在可导航或技能指定的可站立区域。 NavInvalid
坡度 超过技能配置的坡度上限时失败，避免放置物倾斜或穿地。 SlopeTooSteep
视线 / 遮挡 是否需要直视由技能配置决定。需要直视的技能不能隔墙放置。 LineOfSightBlocked
动态状态 施法者死亡、被控制、技能进入 CD、资源不足时，落点结果无效。 CasterStateInvalid

6. 宽容与修正

宽容只解决“差一点”的手感问题，不改变玩家的目标语义。所有修正都需要落在同一支撑面或同一可理解区域内。

策略 行为 适用场景
Reject 非法即失败，不修正。 精准放置、强策略技能、会造成不公平误放的技能。
ClampToRange 超出距离时沿瞄准方向夹到最大距离。 普通地面范围技能。
SnapSameSurface 在同一支撑面内找最近合法点，距离不超过配置容差。 召唤、陷阱、地面圈在边缘附近的手感优化。
UseLastValidPreviewOnly 拖拽预览时短暂保留上一个合法点，释放时仍重新判定。 鼠标或摇杆抖动导致的预览闪烁。
禁止行为：跨层吸附、穿墙吸附、绕过遮挡吸附、把失败改成自动自施法、释放失败仍消耗资源或进入 CD。

7. 技能配置

7.1 PlacementProfile

字段 说明 示例
PlacementMode 技能落点类型。 GroundAOE / Deployable / ProjectileLanding
MaxRange 最大施法距离。 1200 cm
SurfaceMask 允许承载的表面类型。 地面、可站立平台、技能专用区域
FootprintShape 实际占地形状。 点、圆、盒、胶囊
RequireNav 是否要求可导航。 召唤物默认开启，纯范围圈可按技能关闭。
RequireLineOfSight 是否需要施法者到落点无遮挡。 隔墙不能放的技能开启。
HeightPolicy 高低差规则。 同层允许、小台阶允许、跨层失败。
ResolvePolicy 非法或略微越界时的处理。 Reject / ClampToRange / SnapSameSurface
PreviewStyle 预览表现规则。 颜色、边缘、失败提示、吸附标记。

7.2 技能类型默认值

类型 默认规则 程序接入点
地面范围圈 允许距离夹取；需要完整范围可显示；是否需要 Nav 由技能决定。 SkillAreaPreviewDataAsset / SkillAreaSystem
召唤物 必须校验完整占地、导航和阻挡；默认允许同层小范围贴边。 SkillModuleSpawnMobaUnit

投射落点 预览终点、抛物线落点和服务器落点使用同一 FinalPlacement。 SkillModuleLaunchProjectile

8. 预览与反馈

玩家看到的位置必须接近最终结果。发生修正时，预览显示修正后的落点；发生失败时，预览进入非法状态，释放不应产生隐藏的成功或隐藏的消耗。

合法

正常颜色显示完整 footprint，释放时直接使用当前 FinalPlacement。

非法

非法状态要明确，不释放、不消耗；例外必须由技能单独配置。

已修正

吸附不能只移动中心点，还要显示修正后的完整占地范围。

状态 显示 释放结果
合法 使用正常范围圈、召唤轮廓或落点标记。 释放到当前 FinalPlacement。
已修正 显示修正后的最终落点；必要时用细线或边缘提示原始意图和最终点有偏移。 释放到修正后的 FinalPlacement。
非法 范围圈变为非法色，召唤轮廓隐藏或变红，失败原因可供 UI 文案读取。 默认不释放、不消耗、不进入 CD。
预览抖动 拖拽时可短暂保留上一个合法点，避免边缘闪烁。 释放瞬间仍按服务器规则复检。
预览可以更平滑，但不能骗人。玩家松手时看到的是哪里，正式结果就应该尽量落在哪里；如果服务器拒绝，需要返回明确原因。

9. 网络与权威

客户端负责高频预览和输入提交，服务器负责最终落点权威。客户端提交的坐标可以作为预测参考，但不能直接作为最终结果。

数据 要求
客户端提交 AimRay、施法者状态、PlacementProfileId、可选预测落点。
服务器复检 用同一 PlacementProfile 调用 PlacementResolver，输出权威结果。
成功同步 实体生成、范围判定、VFX、音效和日志都使用服务器 FinalPlacement。
失败同步 返回 FailureReason，客户端播放失败反馈，不消耗或按技能规则回滚。

10. 边界案例

边界案例用来验收“系统是否尊重玩家意图”。这些场景不要求每个技能都能成功，但要求成功、修正、失败的口径稳定。

案例 默认结果 验收重点
最大距离外一点点 可沿瞄准方向夹取到最大距离。 预览圈显示在夹取后位置，不在原始非法点闪烁。
同平台边缘轻微越界 可在容差内向平台内侧收缩。 完整 footprint 合法，而不是只有中心点合法。
桥面瞄到边缘，桥下也可放 桥面非法就失败，不自动落到桥下。 不能把同屏可见的另一个合法点当成玩家意图。
楼梯 / 斜坡 坡度和高度差在配置内则成功，否则失败。 召唤物不倾斜穿地，范围圈不漂浮。
墙角 / 门框旁放置 地面技能只认地面；墙面技能需独立 Profile。 不要从一个面偷偷转到另一个面。
抛物线落点被遮挡 按弹道解得到真实终点；如果和预览不一致，需要修正预览或失败。 reticle、落点圈、真实命中点一致。

有限内缩

轻微越界可以收回，同面内缩距离超过阈值后必须失败。

面语义一致

V1 不做墙面放置，但这个案例可以提醒后续扩展时不要提交瞬间换面。

状态变化

角色、召唤物或机关进入占地区时，服务端复检可以失败并回传原因。

11. 验收

优先级 验收项 标准
P0 预览和正式结果一致 平地、边缘、斜坡、高低差场景中，释放结果与玩家松手时看到的最终落点一致。
P0 失败不隐藏消耗 非法落点默认不生成实体、不造成伤害、不进入 CD、不消耗资源；例外必须由技能单独配置。
P0 不跨层吸附 楼上、楼下、断崖、平台边缘不会互相吸附；跨层尝试返回 HeightMismatch。
P0 占地校验有效 召唤物和陷阱不会生成在墙里、角色体内、不可导航区或禁止区域。
P1 边缘手感稳定 同一支撑面内的小距离越界可以贴边，拖拽预览不频繁闪烁。
P1 失败原因可读 日志、UI、QA 能看到明确 FailureReason，不出现只有“失败”没有原因的结果。
P1 网络校正可接受 延迟下客户端预测和服务器结果偏差可控；差异较大时能做失败或校正反馈。
建议 QA 场景：平地、楼梯、平台边缘、上下层平台、窄门、墙角、斜坡、动态阻挡、最大距离边界。

12. 程序备注

当前项目已有技能模块、预览数据和范围系统，落点规则建议作为中间层接入。不要让每个技能自己重复写射线、Nav、碰撞和吸附规则。

模块 接入要求
HeroAbility 组织施法输入和服务器请求，读取技能对应的 PlacementProfileId。
SkillModulePreviewData 预览阶段调用 PlacementResolver，不在表现层重复找点。
SkillAreaPreviewDataAsset 只负责范围表现参数；落点合法性由 PlacementProfile 决定。
SkillAreaSystem 使用已经决议出的 FinalPlacement 创建范围，不再二次改落点。
SkillModuleSpawnMobaUnit 服务端复检后再 Spawn，生成位置以服务器 FinalPlacement 为准。
SkillModuleLaunchProjectile 抛物线和落点指示读取同一结果；飞行命中由投射物逻辑继续处理。

建议改造顺序

  1. 先建立 PlacementProfile、PlacementResolver、ResolvePolicy、FailureReason。
  2. 优先接入地面范围圈和召唤物，这两类最容易暴露预览与正式结果不一致。
  3. 把客户端预览和服务器释放改成调用同一套 resolver。
  4. 再接入投射落点和抛物线终点，统一终点显示和服务器落点。
