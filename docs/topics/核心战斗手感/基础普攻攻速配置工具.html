概述

文件： - Scripts/apply_basic_attack_profile.py

这个脚本用于批量套用一套固定规则： - HeroData.AttackSpeed = 基础显示攻速 - HAD.PlayRate = (1 / 基础显示攻速） * AttackSpeed - Montage AttackIntervalStart.AttackInterval = 1 / AttackSpeed

这样当 AttackSpeed == 基础显示攻速 时： - PlayRate = 1 - AttackInterval = 1 / 基础显示攻速

当前脚本内置了 Steel 示例： - 基础显示攻速：0.625 - 所以： - PlayRate = 1.6 * AttackSpeed - AttackInterval = 1 / AttackSpeed

适用前提

这个脚本假定目标英雄的资源结构和 Steel 这套一致： - HeroData 里有 Unit.Attribute.AttackSpeed - HAD 的 ActivateSkillModules 里有 SkillModuleMontagePlay - 每条攻击 montage 里有一个 AnimNotify_AbilityTrigger - 该 notify 内挂了一个 SkillModuleAttackInterval

它不会处理这些内容： - AttackIntervalStart 的具体时间点 - EndAbilityTimer - Recovery / ABP 收招 - SaveAttack / ResetCombo - 其它非普攻模块 - bAutoReTriggerOnHold（长按持续触发，需手工在 SkillModuleAttackInterval 上勾选）

长按持续触发（自动续触）

如果该英雄的普攻希望支持"按住 LMB 持续输出"(combo 段式连击 或 Sparrow 式充能型连射）， 在 SkillModuleAttackInterval 上手工勾 bAutoReTriggerOnHold = true 即可。

  • 默认 off，既有英雄行为零变化
  • 不需要改动 montage / combo / 攻速换算，本机制只决定"AttackInterval Timer 结束时 是否替玩家自动按一下"
  • 详细语义、被否决方案、约束边界见 [adr-0004]
  • CONTEXT 术语条目见 CONTEXT.md "战斗 · 输入续触" 章节

使用方式

先做： 1. p4 edit 目标 .uasset 2. 确认本地文件不是只读 3. 在脚本顶部 PROFILES 里改目标英雄路径和数值

注意： - 这项目里偶尔会出现 p4 edit 之后 .uasset 仍然是只读属性的情况 - 这种情况下脚本会直接报错并停止 - 先把文件只读位清掉，再执行脚本

建议至少改这几个字段： - hero_data_asset - had_asset - montage_assets - attack_speed_value - play_rate_reference_speed - attack_interval_numerator

常见配置： - 如果你想要 AttackSpeed = 0.625 时 PlayRate = 1 - attack_speed_value = 0.625 - play_rate_reference_speed = 0.625 - attack_interval_numerator = 1.0 - 如果你想要 AttackSpeed = 0.8 时 PlayRate = 1 - attack_speed_value = 0.8 - play_rate_reference_speed = 0.8 - attack_interval_numerator = 1.0

执行方式： - Unreal Editor 里用 File -> Execute Python Script - 或 Python 控制台执行：

exec(open(r"C:/Users/User/Perforce/projectm_likeke_devel/GameProject/Scripts/apply_basic_attack_profile.py", encoding="utf-8").read()）

改别的英雄时怎么填

只要遵守同一套口径： - attack_speed_value - 写进 HeroData 的基础攻速 - play_rate_reference_speed - 你希望 PlayRate = 1 时对应的攻速值 - attack_interval_numerator - 一般填 1.0 - 结果就是 AttackInterval = 1 / AttackSpeed

举例： - 目标：基础攻速 0.7，并且该基础档动画正常速率 - attack_speed_value = 0.7 - play_rate_reference_speed = 0.7 - attack_interval_numerator = 1.0

脚本实际改了什么

HeroData - 改 hero_attribute_datas 里 Unit.Attribute.AttackSpeed 的 BaseValue

HAD - 改 SkillModuleMontagePlay.PlayRate.BaseValue.LevelValueMap

Montage - 改 AnimNotify_AbilityTrigger -> SkillModuleAttackInterval -> AttackInterval.BaseValue.LevelValueMap

限制

这个脚本目前按 Steel 的资产形态写的，后续如果有这些差异，需要再补： - 一个 HAD 里有多段完全不同的普攻模块结构 - 一条 montage 上有多个 SkillModuleAttackInterval - 使用的不是 AnimNotify_AbilityTrigger - AttackSpeed 不是直接驱动 runtime 属性，而是走显示值到内部倍率的二次换算
