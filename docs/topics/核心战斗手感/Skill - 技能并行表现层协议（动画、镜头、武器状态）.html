Presentation Layer Protocol

技能并行表现层协议（动画、镜头、武器状态）

本文件作为《技能释放关系语义系统》的配套专题，专门约束“逻辑允许并行”之后，动画层、镜头、武器姿态和结束恢复如何交接。

适用范围：并行技能组合 项目背景：GAS + HeroAbility + Overlay 重点案例：Twinblast FastFire / Dash / 左手技能

1. 核心原则

并行不是“两个技能都能激活”就算完成。任何标记为并行的技能组合，都必须额外定义动画层、镜头和武器状态的控制权。否则逻辑并行只代表两个 Ability 同时存在，不代表玩家看到的动作一定成立。

边界： 如果没有可用动画层、专用并行动作或镜头恢复策略，正式规则不应标记为并行，应回退为阻挡或打断。

2. 并行类型

并行不是一种单一结果。进入配置前，应先判断这组技能属于哪一种并行，避免把“逻辑可以共存”误解成“动作一定要同时完整播放”。

并行类型 定义 适用例子 主要风险
逻辑并行 两个 Ability 同时 active，但其中一个不争抢动作层，只保留 Buff、状态、伤害或计时。 开火期间开启一个短时强化状态。 策划以为会播动作，动画侧却没有表现。
分层并行 两个技能分别占用不同动画层或身体部位。 右手持续开火，左手释放小技能。 左右手、脊柱、AimOffset、IK 互相覆盖。
运动主导并行 位移、翻滚、Dash 或 RootMotion 拥有移动与主动作，旧技能保留部分表现或只保留逻辑。 FastFire Active 中翻滚。 翻滚动作顶掉开火姿态，或开火姿态破坏位移表现。
武器主导并行 开火、瞄准、武器发射点继续拥有优先级，新技能只能使用不冲突的动作域。 持续开火时插入左手技能。 技能动作需要右手或全身时无法成立。
镜头主导并行 某个技能拥有镜头主控权，另一个技能只能继承或延迟镜头反馈。 Aiming Fire 中 Dash 只给短促相机震动。 镜头被抢走后不能恢复到原来的瞄准状态。
伪并行 / 延迟表现 逻辑先接受输入，但动作延迟到当前不可覆盖段之后播放。 强技能蓄力中缓存一个轻技能动作。 玩家以为立刻响应，实际表现像吞输入。

3. 控制域检查表

控制域 必须回答的问题 Twinblast 示例
全身 / RootMotion 谁拥有位移、翻滚、击退、根运动？是否允许覆盖当前动作？ FastFire + 翻滚时，翻滚通常拥有全身或下半身位移，FastFire 不能再强播会顶掉翻滚的全身 Montage。
上半身 / 武器姿态 开火姿态是否继续维持？瞄准 Overlay 是否保留？枪口和发射点是否还稳定？ 双枪 FastFire 翻滚时，需要“翻滚中持枪/开火”的动作方案，或至少保留武器朝向与发射点，不让开火姿态瞬断。
左右手分层 如果开火占用右手，来袭技能是否只能使用左手层？右手是否继续开火？ 一边开火一边放左手技能时，技能应声明使用 LeftArm / UpperBody 层；右手和武器层继续归开火控制。
镜头 两个技能都想改镜头时谁优先？是否继承当前瞄准镜头？退出时回到哪里？ FastFire 维持 Aiming Camera，Dash 只允许短促镜头反馈；若 Dash 强接管镜头，结束后必须回到 FastFire Aiming 状态。
结束恢复 并行动作播完后，是恢复旧技能循环、进入收招，还是结束旧技能？ 翻滚播完后，如果 FastFire 仍 active，应回到 FastFire 的 Aiming / 开火循环；如果 FastFire 已结束，则走普通移动状态。

4. 资源所有权矩阵

并行配置不能只写“允许并行”，还要写清每个控制域的 Owner。Owner 负责进入、持续、退出和异常恢复，非 Owner 只能读取状态或播放不冲突的附加表现。

资源域 Owner 示例 进入并行时 并行期间 退出恢复
移动 / RootMotion Dash / Roll 暂停旧技能对移动的覆盖。 只允许 Owner 写移动模式、RootMotion、速度曲线。 恢复旧技能移动配置或回到默认移动。
上半身 / Overlay FastFire / Aiming 确认是否保留瞄准 Overlay。 新技能只能占用未被锁定的层。 若开火仍 active，回到开火循环；否则释放 Overlay。
左手 / 右手 Attack owns RightHand，Skill owns LeftHand 声明左右手占用，避免同一手双写。 IK、握把、武器 Socket、施法 Socket 分开维护。 释放对应手部层，不影响另一手持续状态。
武器 / 发射点 ContinuousFire 锁定当前武器姿态和发射点来源。 发射点、枪口特效、弹道方向不能被其它 Montage 顶掉。 停火时统一清理 ContinuousFire、武器特效和开火循环。
镜头 Aiming / Ultimate / Dash 决定新技能是接管、叠加、继承还是不改镜头。 同一时间只允许一个主镜头 Owner，可叠加轻量震动。 回到上一层镜头状态，而不是简单回默认镜头。
输入 当前主技能或新技能 定义按下、松手、确认、取消由谁消费。 避免同一个输入同时触发两个技能的 Press/Release 模块。 退出后重新恢复输入路由和缓存策略。
推荐在技能卡里增加“并行 Owner 表”：每个并行组合至少写出 Movement、FullBody、UpperBody、LeftHand、RightHand、Weapon、Camera、Input 八个域的 Owner 和恢复规则。

5. 表现策略

专用并行动作

例如 RollFire / DashFire Montage，动作本身同时满足位移和开火姿态。适合强英雄特征或高频组合。

分层播放

位移技能占全身或下半身，开火保留上半身 / 武器 / AimOffset，左手技能只占左手层。

逻辑并行、表现抑制

两个 Ability 同时 active，但其中一个暂不播动作，只保留数值、Buff、投射物或状态。

回退为非并行

如果没有可用动画层或镜头恢复策略，正式规则应改为阻挡或打断。

6. 生命周期协议

并行最常见的问题不在进入瞬间，而在退出和恢复。每个并行组合都必须定义完整生命周期。

阶段 必须定义 失败时回退
进入前 当前窗口是否允许并行；目标、冷却、消耗、输入是否重新校验。 阻挡或缓存来袭输入。
进入并行 哪些层保留，哪些层覆盖，哪些层静默；镜头是否接管。 如果关键资源被占用，转为阻挡或打断。
并行持续 Tick、Notify、投射物、RootMotion、Camera Update 的写入顺序。 检测到冲突时结束低优先级表现或结束低优先级技能。
新技能先结束 旧技能是否恢复循环、继续计时、补播过渡或直接进入收招。 恢复失败时清理旧技能表现，避免卡枪、卡镜头。
旧技能先结束 新技能是否继续持有资源；旧技能的特效、镜头、武器状态谁清理。 由旧技能 EndAbility 清理自己拥有的域，不能清理新技能域。
外部打断 眩晕、死亡、沉默、硬控进入时，两个技能按什么顺序结束。 统一走公共清理顺序，不允许单技能残留 ContinuousFire 或 Camera。

7. 验收要求

阶段 验收点
进入并行 原技能动作、镜头、武器状态不瞬断；新技能动作进入清晰。
并行中 动画层不互顶，RootMotion 不冲突，发射点和命中逻辑稳定。
其中一个技能结束 仍 active 的技能能恢复到自己的循环、姿态或镜头状态。
两个技能都结束 角色回到正确移动状态，镜头、Overlay、武器状态全部释放。
并行验收不能只看“能不能激活”。必须录制进入并行、并行中、其中一个技能结束、两个技能都结束四个阶段，确认动作不瞬断、镜头能恢复、武器状态和发射点不丢。
