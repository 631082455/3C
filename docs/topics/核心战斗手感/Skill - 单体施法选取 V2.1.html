Skill - 单体施法选取 V2.1

面向第三人称 MOBA / Hero Shooter 的单体目标技能。系统按候选评分实时驱动 Soft/Armed Target; Holding 期间随评分跟手跳转（β 模式）；Release 时取当时胜出者结算。

需求方：战斗 / 3C 策划 实现方：战斗程序 / UI / 网络 V2.1 - β 翻案版 已落地 M1–M5
V2 → V2.1 关键变化:V2 grilling 后 α 模式被推翻为 β 模式 （Armed 不再"按下即锁死"，随候选评分跟手跳转）。多条"禁止行为"被软化或删除；ResolvePolicy 从 5 个缩成 2 个； 服务器验证策略明确为 300ms 时间窗容忍（不做 full lag-comp）。关联决策记录： ADR-0001不默认 fallback 自施法 ADR-0002β 模式 ADR-0003服务器 300ms 容忍

1. 背景与目标

单体目标技能既要顺手，也要可控。系统在前段持续维护 Soft Target、给 UI 提示、减少空放； Holding 期间允许 Armed Target 随评分跟手跳转，降低"我准星动了但系统没跟上"的不一致感。

核心规则（V2.1 修订）

系统按候选评分实时维护 Soft/Armed Target；原目标失效时按评分规则自然让位给次佳候选。 不能自动切成自施法(AllowSelfFallback 必须技能级显式 opt-in)。

2. 本版范围

本版要做（已完成）

  • 候选实时维护：空闲（且有驱动源时）+ Holding 期间持续 20Hz 评分跑候选选取
  • β 模式：Holding 期间 Armed 随候选评分跟手跳转
  • 评分稳定：LockRetainRadius / LockProtectTime / SwitchCooldown / Grace 加分
  • 技能级配置：TargetSelectionProfile + ResolvePolicy 表达差异
  • UI 三态（Soft / Armed / SelfCast）+ 屏幕空间 widget
  • 服务器 300ms 时间窗容忍验证（ADR-0003）
  • Alt+技能键 全局自施法（立即施放，跳过 Hold 蓄力）
  • 右键全局取消 Holding（无消耗 / 无 CD）
  • Editor 校验：每英雄至多一个驱动源

本版不做

  • 不引入连续光束 / channeling 类技能（本版仅覆盖按住-蓄力-确认范式）
  • 不引入目标循环按键(Tab 切换) — 纯瞄准驱动
  • 不覆盖范围技能、地面点选、锁敌相机、辅助瞄准完整方案
  • 不做 full lag compensation rewind(ADR-0003)

3. 术语

术语定义使用边界
候选目标 通过硬过滤的可参与评分 Actor 进入评分前必须先过硬过滤
硬过滤 阵营、死亡、世界距离、ScreenCone、LOS、技能目标类型、可选中性 不满足直接出局，不参与评分
Soft Target 空闲（非 Holding）阶段的当前评分胜出者 UI 弱（灰白 L 括号）；仅在有驱动源 + 技能可用时显示
Armed Target Holding 阶段的当前评分胜出者(随评分跟手，不锁死) UI 强（队伍色 L 括号）；Release 时打它；评分胜出者切换 → 跟手跳转
驱动源 (Drive Source) 英雄装备的单体技能中，被标记为 bDrivesIdleSoftTarget=true 的那一个 每英雄至多一个 （Editor 校验）；决定空闲时是否显示 Soft Target
Grace 加分 候选短暂失效（出 LOS / 出范围 / 出 ScreenCone）时，在评分中保留较高加分一段时间 防单帧抖动；不用于"保留原 Armed"；死亡 / 阵营反转 / NotTargetable 不享受
LockRetainRadius 已锁定目标维持的评分加成半径（屏幕空间） 防边界抖动
LockProtectTime / SwitchCooldown 评分胜出者改变后的强制冷却 防 A/B 抖动
ResolvePolicy Release 时候选池空的收口规则 仅 2 个：NoCastOnEmpty / AllowSelfFallback
SelfCast Alt+技能键 触发的显式自施法，或 AllowSelfFallback 触发的回落自施法 UI：自蓝 L 括号挂自己

4. 行为规则

阶段系统行为
空闲 + 有驱动源 + 技能可用 20Hz 持续候选采集 + 评分；Soft Target = 评分胜出者；widget 挂在 Soft Target
空闲 + 无驱动源 完全不算 Soft；屏幕 0 框
空闲 + 驱动源 CD / 沉默 / 无蓝 / 未学 不显示 Soft 框（HUD 技能图标已表达不可用）；Invalid 无独立 UI
点按快放 按下瞬间取当前候选胜出者 = Armed，立即 Resolve；不进入 Holding
按下进 Holding 用该技能 Profile 接管候选采集；Armed = 当前评分胜出者；UI 升级为队色 L 括号
Holding 中目标失效 / 评分变化 Armed 自然跟手跳转到次佳候选（死亡 / 失 LOS / 失范围 / 评分被甩超 LockRetainRadius）
Holding 中玩家主动甩枪 等同于上一条；评分变化驱动，无"玩家意图检测"特殊分支
Holding 中候选池完全为空 Armed 暂时无，框不显示；Release 走 ResolvePolicy
Release（松开技能键） 取当时的 Armed Target（或候选池空 → ResolvePolicy）。Client 打包成 EventData 走现有 RPC
Cancel（右键） Holding 中止；无消耗 / 无 CD / 无动画 / 无 UI 反馈；框消失
Alt+技能键（SelfCast） 跳过 Soft/Armed 评分；立即施放在自己身上（跳过 Hold 蓄力等待）；走 SelfCast UI
Alt+技能键 但该技能 AllowedTargetTeam 不含 Self 静默吞掉；不响应、不报错、不给 HUD 提示

红线（收窄后）

  • 系统不得自动切成自施法(必须 AllowSelfFallback 显式 opt-in，且 UI 必须显示"将自施法")
  • 玩家显式输入（Alt+键、右键取消）优先级永远高于评分自动决策
  • 服务器不擅自替换 client 上报的 ArmedTarget（只验证 / 拒绝）

V2 → V2.1 已删除的旧禁止行为（被 β 翻案）

  • 目标短暂丢失时偷偷换成另一个目标 — β 下就是该跟手跳
  • Grace 结束后自动跳到下一个目标 — β 下次佳自然上位
  • 用"最近友军/最近敌人"替代已锁存目标 — β 下是正常机制
  • 目标失效时自动发明新目标 — β 下次佳就是"新目标"

7. ResolvePolicy

β 模式下"目标失效"实际只剩候选池完全为空这一种情况。原 V2 doc 5 个 policy 缩成 2 个。

策略行为适用
NoCastOnEmpty （默认） 候选池空 → 啥也不发生：无消耗 / 无 CD / 无施法动画 / 无 SkillModule / 无 UI / 无音效 等价于"按键失效，系统当你没按过"。所有非"防御性资源型"技能用这个
AllowSelfFallback (opt-in) 候选池空 → cast 在自己身上，走 SelfCast UI；正常消耗 / 进 CD / 播动画 / 触发 SkillModule 防御性资源型友军技能（治疗 / 护盾）的明确设计意图
V2 → V2.1 已删除的 policy:
  • FailOnInvalid → 合并到 NoCastOnEmpty（消耗 / CD / 动画 / 反馈全去掉）
  • CancelCast → 跟 NoCastOnEmpty 等价
  • RevalidateSameTarget → β 下"原 Armed"概念不持续
  • ExplicitSelfCastOnly → Alt+键全局机制，不需要 policy 字段
决策背景见 ADR-0001 不默认 fallback 自施法。

8. UI 与网络

8.1 UI 三态 + L 括号

统一形态：四角 L 形括号，贴 target 在屏幕上的投影包围盒（远小近大）。 同一时刻只有一个 widget 显示（挂在当前 Soft/Armed/SelfCast 对应的 actor 身上）。

图 8.1 · 三态色板：Soft 灰白 / Armed 队色 （友绿+敌红） / SelfCast 自蓝

Invalid 不存在独立 UI 态：技能 CD / 沉默 / 无蓝 / 未学时，Soft 框直接不显示。 HUD 技能图标已表达不可用状态。

β 跳转视觉信号:widget 瞬移（从 A actor 消失，在 B actor 出现）— 框的位置变化即 motion event, 玩家眼睛即使在战场中央也能感知。不加跳转音效，纯靠视觉。

屏幕中心准星与本系统完全无关：准星只服务普攻/枪械，不随 Soft/Armed 状态变色或变形； 单体目标状态完全靠 L 括号传达，准星是独立信道。

ScreenCone 边界：不画（若未来 playtest 反馈"新人不懂 Armed 跳"再开 V3 模式）。

Release 失败 / 右键取消 / 技能名 label：全部无 UI 反馈（极简风格）。

8.2 网络与权威 ADR-0003

事项要求
客户端20Hz 跑候选评分，预测 Soft/Armed UI;Release/Confirm 时打包 Armed Target 进 FGameplayEventData 走现有 ServerConfirmHoldingWithTimeStamp RPC
服务器验证 Armed Target 合法性，采用 300ms 时间窗容忍模式
容忍项Dead / OutOfRange / NoLOS / OutOfScreenCone（300ms 内合法过即放行）
不容忍项InvalidTeam / NotTargetable / 技能 CD / 沉默 / 蓝量（永远按服务器当前判定）
容忍允许但执行时已失效静默 fail；客户端不显示失败反馈；玩家感知"我以为打到了"但不被打脸
反作弊ClientTime 须做 sanity check(abs(ClientTime - ServerTime) < ping * 2)，否则拒收
回放 / 观战记录最终决议目标、ResolvePolicy 触发（若有）、自施法标记
图 8.2 · 服务器 300ms 容忍窗：client 视角的 cast 在 server 时间窗内合法即放行（实际执行仍可能 fail）

9. 技能示例：Muriel Shield

项配置
AllowedTeamsFriendly （+ FriendlySummon 可选）
RangeCm1200（12 米）
ScreenConeRadius0.30（屏幕中心半径 30%）
LineOfSightRuleRequiredWithGrace
LostGraceTime0.20s
SwitchCooldown0.12s
bDrivesIdleSoftTargettrue（每英雄至多一个）
InputModeHold（按住 + LMB 确认）
ResolvePolicyAllowSelfFallback（空旷场地自动给自己）

典型流程

  1. Muriel 走动 → 屏幕中央友军身上自动出现灰白 L 括号（Soft）
  2. 按下 E → 框升级为绿色 L 括号（Armed = 该友军）
  3. 友军拐过墙 → Armed 自然跳到旁边友军 B（框瞬移到 B 身上）
  4. 松开 E + LMB 确认 → 护盾打到 B 身上
  5. 或：按 E 后右键 → Holding 取消，无消耗
  6. 或：Alt+E → 跳过 Hold，护盾立即上自己
  7. 或：Shield 进 CD / 中沉默 → Soft 框消失，直到可用为止

10. 验收

优先级验收项通过标准
P0β 跟手Holding 期间 Armed 跟随当前评分胜出者跳转；原目标死/出 LOS/出范围 → 自然让位次佳
P0候选池空收口候选池完全为空时 Release → 走 ResolvePolicy（默认无反馈 / opt-in 走 SelfCast）
P0自施法不擅自没标 AllowSelfFallback 的技能，候选池空时 Release 绝不自动 cast 自己
P0UI 三态正确Soft（灰白） / Armed（队色） / SelfCast（自蓝）；Invalid 不显示框
P0不可用即隐藏驱动源 CD / 沉默 / 无蓝 / 未学时，Soft 框立刻消失
P0服务器容忍跨服 ping 150 玩家可正常释放刚死亡 < 300ms 的目标；超 300ms 失败；静默 fail
P0输入优先级Alt+键 / 右键取消优先于评分自动决策
P1Profile 数据化至少两个技能复用同一 Profile，仅改技能卡引用
P1Editor 校验同英雄 2 个 bDrivesIdleSoftTarget=true 报错；AreaPreviewTargetOrSelf 在 dropdown 中不可见
P1Debug 可解释Targeting.DebugDraw 1 显示候选框 + 名字，便于内部排查

11. 程序备注

实现层次（策划视角的 5 个功能模块）

  • M1 目标自动框选 + 显示：空闲时框跟着我看的人走
  • M2 完整单体施法闭环：按下蓄力 + 跟手 + Release + 右键取消
  • M3 自施法 + 防御性技能 fallback:Alt+键 + AllowSelfFallback
  • M4 联机 + 高 ping 容忍：服务器 300ms 容忍窗
  • M5 Editor 规范 + production UMG widget

数据/状态拆分

UTargetSelectionProfileDataAsset
  AllowedTargetTeam / AllowedUnitTypeTags
  Range / ScreenCone / LineOfSightRule
  ScoreWeights / LockRetainRadius / LockProtectTime / LostGraceTime / SwitchCooldown
  ResolvePolicy
  TargetIndicatorWidgetClass / TargetIndicatorOffset

UTargetSelectionComponent(挂 AMatchPlayerController)
  ActiveProfile      (空闲=驱动源,Holding=该技能 Profile)
  CurrentTarget      (Soft / Armed)
  State              (Idle / Holding / SelfCast)
  IdleDriveAbility   (Discovery 找到的驱动源 ability)
  GraceMap           (短暂失效候选的 timestamp)
  TargetIndicatorWidget (UWidgetComponent,挂在 CurrentTarget 上)

约束

约束说明
统一入口目标搜索、评分、Holding 状态机、Release 决议都走 UTargetSelectionComponent；技能蓝图不重复实现
技能只给配置通过 TargetSelectionProfile + 技能卡字段表达差异；不写 C++ 找目标
手动优先Alt+键 / 右键取消 / 玩家显式甩枪优先级高于自动评分
反作弊ClientTime sanity check；不容忍项严格 server-now 判定
