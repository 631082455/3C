战场交互物机制扩展与 Muriel Bomb 对标

这份文档不再把 Touch Pickup、Step Trigger、 Carry 放在同一层级，而是先收成主干三类： 怎么触发、怎么结束、敌方怎么处理。 如果对象会进入携带态，再额外挂一个 特殊状态扩展。

当前入口：Muriel Bomb 当前语义：接触触发 + 消耗 + 上 Buff 目标：为后续拾取物、部署物、携带物共用中层

案例

从具体技能条目倒推机制

这里不做泛泛的“某游戏也有交互物”罗列，只抓那些真正能反推程序规则的技能条目。 每个案例都只保留：技能最简单的作用、它怎么触发、会变成什么、玩家还能怎么处理它。

Overwatch

Illari - 治疗炮台

  • 技能简述：放一个治疗炮台，自动给范围内队友回血。
  • 对象落地后持续存在，自动对范围内友军治疗。
  • 敌方可破坏，主人可重新部署。
  • 推导：部署物不能只会“存在”，还要有血量、回收、重放、被反制。
Overwatch

Lifeweaver / Symmetra 交互装置

  • 技能简述：一个是接触后原地激活的平台，一个是点一下就传送的装置。
  • 花瓣平台：接触平台后原地激活。
  • 传送门：OnInteract → Teleport。
  • 推导：接触触发和交互触发都可能依赖 overlap，但结果语义完全不同。
Marvel Rivals

Rocket - B.R.B.

  • 技能简述：放一个支援装置，它会持续提供团队价值，并且可回收。
  • 装置持续存在，并与周期产出或支援效果绑定。
  • 可主动回收，体现“回收返还”也是正式玩法。
  • 推导：对象存在、产出子对象、回收返还，应是中层能力。
League of Legends

LoL 典型交互物

  • 技能简述：这里混合了点一下生效、碰到就吃，以及“被攻击处理”的反制语义。
  • 灯笼：OnInteract → Teleport。
  • 音符：OnTouch → Consume。
  • 桶子 / 种子：不是顶层触发方式，而是对象可被攻击处理，并允许敌方否决。
  • 推导：生成、接触、交互是触发入口；攻击更适合放在“敌方处理方式”里。
Deadlock

Deadlock：魂瓮 / 炮台

  • 技能简述：一个是典型携带目标物，一个是典型可放置炮台。
  • Spirit Urn 暴露的是 Carry 态边界：死亡、传送、断线、能力封锁。
  • Turret 暴露的是落点、贴墙、放置失败与回收问题。
  • 推导：状态机、失败兜底、特殊位移兼容都不能后补。
机制

相关程序机制

机制层 建议内容 为什么必须有
对象基类 SkillWorldInteractActor：飞行、落地、静止、滑行、墙停、回收。 没有统一宿主，后续所有技能都会把生命周期写回单技能蓝图。
触发方式 OnSpawn / OnTouch / OnInteract 定义怎么触发；OnTouch 表示进入对象或区域即触发，OnInteract 可做成立即确认或读条完成。
结束方式 DestroyOnUse / ExpireOnLifetime / Recall / CompleteAtDestination 定义对象怎么退出场上、被召回，或在系统层面完成结算。
敌方处理方式 None / EnemyTouchOrInteractRemove / Attackable 定义敌方能不能通过接触、交互或攻击来拆除、否决或摧毁对象。
特殊状态扩展 EnterCarryState / DropOnDeath / AllowDiscard 只有对象会进入携带态时再挂这层，不要塞回基础分类里。
权限过滤 InteractionFilter：阵营、单位类型、自拾、OwnerIgnoreWindow、ArmDelay。 Bomb、灯笼、桶子、种子，本质都先过过滤层。
状态机 Spawned → Traveling → Armed → Eligible → Claimed → End 没有正式状态机，异常最后都会碎成蓝图 if/else。
效果包 InteractionEffectPackage：给拾取者、给周围友军、给敌方、给场景。 Bomb 这种“拾取者 + 周围友军”已经不是单目标效果。
失败兜底 AFK、地图外、异常附着、长时间无人交互重置。 否则对象会卡场，严重时直接锁玩法。
关于 Carry： 文档里的 Carry 不是“核心位”含义，而是 EnterCarryState：对象被拾取后，角色进入“携带态”，并由该状态继续约束技能、死亡、掉落与重置规则。
Muriel Bomb

Muriel Bomb 现在该怎样归类

按你们当前的 Bomb 规则，最合理的第一步不是直接做成完整 objective， 而是先作为一个语义清楚的样例落地：接触触发 + Consume + ApplyBuff。

维度 Bomb 当前推荐定义 未来可扩什么
对象基类 SkillWorldInteractActor：飞行、地面滑行、墙停、停留 6s、超时回收。 以后所有拾取物、部署物、携带物都复用这层。
触发方式 OnTouch，并带 OwnerIgnoreWindow 和 ArmDelay。 以后可扩到 OnInteract，比如按键确认或读条交互。
拾取后效果 Consume + ApplyBuff：给拾取者和周围友军结算。 以后可扩到 EnterCarryState 或 SpawnChild。
结束方式 触发后立即结束；若无人拾取则超时回收。 以后也可扩到系统层面结算，或被主人主动召回。
敌方处理方式 当前敌军不可拾取；掉出地图、异常静止、重复触发、AFK/断线都要 Reset。 以后可扩到敌方拆除、敌方攻击摧毁或敌方否决。
特殊状态扩展 当前不进入携带态，也不需要主动丢弃或死亡掉落。 如果以后做魂瓮式玩法，再挂 EnterCarryState / DropOnDeath。

Bomb 最值得先补的 P0

  • 对象基类：飞行、停留、拾取、回收不要再堆在单技能蓝图里。
  • InteractionFilter：阵营、自拾、OwnerIgnoreWindow、ArmDelay 配置化。
  • InteractionEffectPackage：拾取者和周围友军效果统一打包。

Bomb 值得预留的 P1

  • Recall / Reclaim：可回收、可返还部分冷却。
  • Enemy Counterplay：以后可扩到摧毁、否决、打断。
  • Spawner Variant：以后做周期产出包时不用重来。

不要继续堆在单技能蓝图里的东西

  • Owner 特例。
  • 拾取异常重试。
  • 墙停 / 滑行 / 超时回收边界。
  • 范围 Buff 广播。
  • 对象销毁 / 回收的 Cue 分支。
建议的程序结论： 优先级最高的工作不是继续给 Bomb 加特例，而是先把 SkillWorldInteractActor + 触发/结束/敌方处理规则 + InteractionEffectPackage 做成正式中层。
