# 产出区规范

## 目录组织

```
output/
├─ examples/          # 本库自带样板（勿改，作为规格参照）
├─ characters/<角色名>/   # 逐角色 3C 方案
├─ systems/<系统名>/      # 系统级方案（镜头体系、移动预设体系…）
└─ audits/<对象>-<日期>/  # 审计报告与调参建议
```

## 命名规范

- 文件名：小写-连字符，`<对象>-<主题>-3c.md`（如 `frost-archer-dash-3c.md`）
- 审计报告带日期：`movement-audit-2026-07.md`
- 同一方案多轮迭代不建新文件，文内版本号 + 变更记录

## 规格要求

- 完整方案：必须走满 `templates/blank-3c-design-doc.md` 8 章（对照 `examples/blink-dash-3c.md`）
- 精简产出（调参建议/快速审计）：允许只含目标、清单核对、参数改动表三部分，但生命周期与冲突审计不可省
- 任何含参数的产出：参数表五列齐全（默认/范围/后果/生效链），预设先行

## 保密提醒

`output/` 会进 git。**含公司项目真实数据的产出放 `knowledge-local/`（已 gitignore），不放这里。** 公开仓库中的产出只允许：虚构项目、公开游戏分析、脱敏后的通用方法演示。
