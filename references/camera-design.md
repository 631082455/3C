# 镜头设计（Camera）

覆盖：构图、跟随、镜头动效层、优先级仲裁、防晕。适用于第三人称/俯角/越肩等实时战斗镜头。

## 1. 镜头职责的优先级排序

战斗镜头的职责，冲突时按此序让步（后者让前者）：

1. **信息**：玩家角色 + 当前主要威胁同屏、可辨
2. **操作参照**：镜头运动不能破坏玩家对方向/距离的判断
3. **反馈**：命中/受击/技能的镜头强调
4. **演出**：情绪、气势、风格化构图

大量镜头争议本质是把 4 放到了 1 前面。评审时先问"这个效果损失了什么信息"。

## 2. 镜头系统的资产化结构（推荐形态）

把镜头行为拆成两层：

- **基础镜头（Rig）**：构图参数（距离、俯仰、偏移、FOV）+ 跟随参数（位置阻尼、死区、前瞻）。逐游戏状态一套（常规/瞄准/死亡观战…），状态切换带过渡时间与曲线。
- **镜头动效（Motion Effect）**：叠加在基础镜头上的临时效果——震屏、冲击方向震、推拉（dolly）、FOV 脉冲、位置偏移、时间缩放联动。**每个动效是一个可配置资产**，而非散落在技能代码里的调用。

动效资产的必填字段：

| 字段 | 说明 |
|---|---|
| 类型 | 震动 / 推拉 / FOV / 偏移 / 复合 |
| 优先级 Band | 见 §3 |
| 时长与曲线 | 起止包络（Attack/Sustain/Release），禁止无限时长除非声明停止方（M6） |
| 强度与衰减 | 含随距离衰减（受击者不是本机玩家时） |
| 结束条件 | 定时 / 事件驱动（事件驱动必须指认停止方） |
| 死亡/打断清理 | 必填，资产校验器应强制（M4） |
| 允许的操作权状态 | 含旋转分量的动效必须限定状态（M1） |

## 3. 优先级仲裁：Band 模型

镜头动效不允许 last-wins（M3）。推荐四带模型：

| Band（低→高） | 内容 | 同带混合规则 |
|---|---|---|
| Ambient | 呼吸、待机微动 | 权重叠加 |
| Feedback | 命中震、受击震、脚步震 | 权重叠加，设总强度上限（防多段命中叠爆） |
| Skill | 技能施放的定向推拉/FOV | 同带取最高权重者，新效果软过渡接管 |
| Cinematic | 处决、大招演出、死亡 | 独占，需声明归还协议（M2） |

跨带规则：高带激活时低带**衰减而非硬切**（乘一个 0~1 的抑制系数），归还时反向过渡。

**审计要求**：新增动效时列出同 Band 现有全部效果，说明叠加/接管行为；跨 Band 说明抑制表现。

## 4. 跟随设计要点

- **位置阻尼各轴独立**：横向可以紧（操作参照），纵向宜松（跳跃/击飞不拽镜头），前后中等。
- **死区 + 软区**：死区内不动，软区内按比例追，出软区硬追。死区大小与角色移速正相关。
- **前瞻（look-ahead）**：朝移动/瞄准方向偏移构图。前瞻量变化必须走阻尼，输入抖动会直接放大成镜头抖动。
- **位移技能的跟随策略**（重点，单独设计）：
  - 短距瞬移（闪现类）：镜头**不跟隨轨迹**，直接软过渡到终点构图（150~250ms），保落点确定性。
  - 冲刺（dash）：跟随但阻尼临时放宽 + 轻微 FOV 外扩，强化速度感；结束时 FOV 回收要慢于位移结束（约 1.5~2 倍时长），否则有"撞墙感"。
  - 超远位移/全图技能：进 Cinematic 带处理，含完整归还协议。

## 5. 防晕清单

- 旋转是最强晕源：遵守 M1 的旋转预算。
- FOV 变化限速：单次变化 ≤ 基础值的 15%，变化速率封顶；禁止高频 FOV 抖动。
- 震屏用**位置震**优先于**旋转震**；旋转震幅度上限约 0.5°，且时长 ≤ 200ms。
- 所有战斗镜头动效提供全局强度缩放（辅助功能选项），0 时信息类构图变化保留、装饰类动效归零。
- 镜头永不穿插角色模型；碰撞回弹用弹簧而非硬切。

## 6. 常见镜头问题 → 排查方向

| 症状 | 优先排查 |
|---|---|
| 偶现镜头异常、无稳定复现 | 是否存在 last-wins 旁路绕过了仲裁（M3）；找出所有直接写镜头的调用点 |
| 死亡/重生后镜头不对 | 动效死亡清理缺失（M4）；Cinematic 带归还协议缺失（M2） |
| 配了参数没效果 | 沿生效链逐层验证（M10）；确认不是死配置（M14） |
| 镜头"晕" | 旋转分量审计（M1）+ FOV 变化速率 + 阻尼频率是否与移动频率共振 |
| 位移技能后镜头"甩" | 跟随阻尼在位移结束瞬间未做软放宽；或前瞻量突变 |
