# 反模式自检清单（Anti-Patterns）

Phase 4 必过。每条给出：症状 → 为什么错 → 替代做法。方案文档中逐条标注"不涉及 / 已规避（如何）"。

---

### AP1 — Last-wins 镜头/速度覆盖
**症状**：新效果直接写目标值，谁最后触发谁生效。
**为什么错**：行为取决于触发时序 → 网络环境下不可复现的偶现异常（M3）。
**替代**：进优先级 Band/乘区仲裁；封死直接写入口。

### AP2 — 无清理路径的持续修饰
**症状**：加了移速/镜头/姿态修饰，只写了施加没写移除，"反正 buff 结束会掉"。
**为什么错**：死亡、打断、异常路径不走正常结束 → 永久残留（M4）。
**替代**：过八问生命周期清单；死亡清理默认必配。

### AP3 — 表现层承载逻辑
**症状**：周期伤害挂在特效 Cue 上、判定挂在动画 Notify 上。
**为什么错**：表现层不保证执行（裁剪/LOD/打断）→ 偶现数值不一致（M7）。
**替代**：逻辑进权威层周期器；表现层只听事件做表现。

### AP4 — 预览与执行参数源分裂
**症状**：指示器一套参数，判定另一套；或指示器写死数值。
**为什么错**：所见非所得，最伤玩家信任（M8）。
**替代**：单一参数源，预览/执行都从它取。

### AP5 — 参数海无预设
**症状**：机制上线暴露 20+ 原子参数，"策划可以自由调"。
**为什么错**：没人会调，所有人抄第一份配置；调的人产出不可控（M9）。
**替代**：预设档位 + 少量微调项；原子参数留给预设的制作者。

### AP6 — 用结果量判断意图
**症状**：用 Velocity 方向判断前进/后退，用当前位置推断玩家目标。
**为什么错**：结果量含系统反馈（击退/矫正），受迫时判断反；预测环境两端不一致（M11/N2）。
**替代**：读输入与加速度。

### AP7 — 操作中旋转玩家镜头
**症状**：技能命中/演出效果带镜头旋转，玩家正在操作时触发。
**为什么错**：晕 + 失控感，旋转预算应随操作权收缩（M1）。
**替代**：操作中只用位置/FOV/震动分量；旋转演出限死亡/过场。

### AP8 — 夺取控制权无归还协议
**症状**：演出/控制夺取输入或镜头，归还逻辑"应该会自动回去吧"。
**为什么错**：异常路径（死亡/打断/重连）不归还 → 卡死类 bug（M2）。
**替代**：归还时机、归还状态、强制归还路径三件套写进方案。

### AP9 — 自控效果取消自身
**症状**：自晕/自缚技能上线后表现残留、效果不清理。
**为什么错**：自施控制走通用打断逻辑把技能自己取消，后续表现成孤儿（M5/M6）。
**替代**：自控技能进"不打断自身"白名单，列入新技能 checklist。

### AP10 — 动画阻塞输入响应
**症状**：为了动画完整性，起步/转身/收招动画期间输入不生效。
**为什么错**：牺牲响应换表现是 3C 的方向性错误——表现可以追，响应丢了就是丢了。
**替代**：先响应后表现；动画用过渡/叠加层追上状态。

### AP11 — 在死配置上叠新机制
**症状**：新机制复用"现有配置项"，但该配置项实际早已无代码读取。
**为什么错**：方案建立在不存在的地基上，实施时才发现全要重做（M14）。
**替代**：方案前审计相关配置的真实生效状态（生效链走查，M10）。

### AP12 — 体验目标不可验证
**症状**：方案目标写"手感更好""更爽"。
**为什么错**：无法验收，评审变成口味之争（M13）。
**替代**：每个目标配参照物 + 可执行验证法。

### AP13 — 全局硬切代替灰度
**症状**：手感类改动（速度模型、缓冲规则）直接全局替换旧行为。
**为什么错**：手感改动影响所有角色的肌肉记忆，问题面即全量玩家。
**替代**：参数默认值=旧行为，逐角色/逐模式开启；观察后再切默认。

### AP14 — 忽略非本机视角
**症状**：反馈/表现只按"我是攻击者"调，旁观者与受击者视角没人看。
**为什么错**：MOBA/多人游戏 90% 的命中你是旁观者；旁观视角全强度震屏是灾难。
**替代**：攻/受/旁观三视角参数分开配并分别验收。

### AP15 — 主准星跟随真值治保持型偏移
**症状**：镜头演绎挪了屏幕中心，于是让准星 UI 每帧跟到真实瞄准方向的投影处，"这样准星永远诚实"。
**为什么错**：准星是眼睛的锚，UI 自行移动读作故障——逻辑全对，体验是"乱飘"（M15）。CS2 Follow Recoil 职业不开早已预警。
**替代**：保持型偏移用瞄点锁定环绕（look-at 反解）合法化；需要显示真值时用次级半透明标记，主准星钉死屏幕中心。（出处：CASE-01）

### AP16 — 实验功能的代码默认值设为"开"
**症状**：实验性开关在 C++ 基类里默认 true，"反正资产里能关"。
**为什么错**：直接继承基类、没显式配置的资产**全部暗中启用**，未来某天触发条件凑齐时集体出问题，且无人知晓根因。
**替代**：代码默认关，资产显式开；实验开关列入代码审查清单。（出处：CASE-01）

### AP17 — 双重折扣
**症状**：资产端把幅度烤小，引用端强度再打一次折，效果直接不可感知，"为什么配了没反应"。
**为什么错**：标定值失去唯一性后，任何一端看起来都"配了合理的数"，问题藏在乘积里。
**替代**：标定值唯一——资产即基准（强度=1.0 时就是设计值），引用端只做围绕 1.0 的微调。（出处：CASE-01）

### AP18 — 信动画描述定关键帧
**症状**：逻辑触发帧按动画文件名/描述/口头约定来配（"出手大概在 0.37"）。
**为什么错**：动画描述与实际表演帧经常错位，错位量肉眼难辨但判定可感（CASE-01 实测抓到 0.03 vs 0.37 的发射帧错配）。
**替代**：关键帧一律实测定（逐帧记录/断点），实测值写进配置注释。
