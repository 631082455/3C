# 打击感（Hit Feel）

覆盖：打击感的时序分解法、反馈件清单、强度分级、多段命中处理。

## 1. 方法：先分解时序，再堆反馈件

"打击感不足"不可直接调参。先把一次命中拆成时间轴，逐段检查缺什么：

```
T-n  前摇预示（蓄力感：动画拉弓、音效蓄、特效聚气）
T0   判定帧
T0+  命中确认包（0~50ms 内密集释放）：
     ├─ 攻方顿帧（hitstop）
     ├─ 受方顿帧 + 受击动画
     ├─ 命中特效（火花/刀光交点）
     ├─ 命中音效（与挥空音效必须可区分）
     ├─ 镜头反馈（Feedback 带震屏，方向性优于随机）
     └─ 材质反馈（受击闪白/描边）
T0++ 结果表现（击退位移、伤害数字、UI 反馈）
尾段 收招与残留（烟尘、地面痕迹、音效尾巴）
```

诊断口诀：**预示不足 → 打击"轻"；确认包不足 → 打击"糊"；结果不足 → 打击"没分量"**。三类问题药方完全不同，先确诊再开方。

## 2. 反馈件参数要点

### 顿帧（Hitstop）
- 攻受双方可不对称（攻方短受方长，攻方保流畅、受方强调受创）。
- 只冻结双方角色（动画/移动），不冻结镜头与全局时间；范围技能命中多目标时攻方顿帧**不叠加**（取最大或首个）。
- 量级参考：普攻 30~60ms，重击 80~120ms，处决级 150ms+。竞技向整体下调，宁轻勿粘。
- 网络注意：顿帧是纯表现，各端本地播，**不进权威模拟**（否则位移判定被表现污染，见 `netcode-3c.md`）。

### 震屏
- 走镜头动效层 Feedback 带（`camera-design.md` §3），**不要直接写镜头**。
- 方向性震（沿命中方向的位置脉冲）优于随机震；量级与顿帧同步分档。
- 本机是攻方/受方/旁观者，三种视角强度分开配（旁观者大幅衰减）。

### 受击动画
- 至少 4 向（前后左右受击），按命中方向选择；轻中重三档对应"上身抖动 / 硬直步退 / 倒地"。
- 用叠加层播轻受击（不打断移动），硬直档才占全身层——受击反馈不应过度惩罚移动手感。

### 材质与特效
- 受击闪白：全局默认（时长 80~120ms + 衰减曲线）+ 逐特殊情况覆盖；连续命中时重触发而非叠亮度。
- 命中特效挂**命中点**而非角色原点；朝向取命中法线或攻击方向。

## 3. 强度分级体系

把上述所有反馈件的量级打包成**命中等级预设**（M9）：

| 等级 | 适用 | 顿帧 | 震屏 | 受击 | 音效层级 |
|---|---|---|---|---|---|
| L1 轻 | 普攻平砍、持续伤害跳字 | 短/无 | 无 | 叠加抖动 | 轻 |
| L2 中 | 技能命中、重普攻 | 中 | 小方向震 | 步退 | 中 |
| L3 重 | 控制命中、爆发技能 | 长 | 中震+FOV微脉冲 | 硬直 | 重 |
| L4 演出 | 处决、大招核心命中 | 特殊 | Skill/Cinematic 带 | 专属动画 | 专属 |

技能配置时只选等级 + 覆盖个别项，不逐技能从零堆反馈件。

## 4. 多段命中与高频命中

- 多段技能（连发/引导）：首段用设计等级，后续段自动降 1~2 级，最后一段可回升收尾——否则全程 L3 会"糊成一团震"。
- 高频 AOE 多目标：镜头 Feedback 带设总强度上限（`camera-design.md` §3）；音效做同类限流（同 50ms 窗口内合并）。
- 持续伤害（DoT）跳字默认 L1 以下（无顿帧无震屏），**周期性结算逻辑本身在权威层**（M7），表现只按节拍点缀。

## 5. 常见打击感问题 → 排查方向

| 症状 | 优先排查 |
|---|---|
| "打击轻/像挠痒" | 时序分解：通常缺确认包中的顿帧+音效区分度；先加 L 级预设再谈特效 |
| "糊/黏" | 顿帧过长或多目标叠加；多段命中未降级 |
| "震得晕" | 震屏走了旋转震/未设上限；改方向性位置震（`camera-design.md` §5） |
| 偶现无反馈 | 反馈挂在表现层事件（Cue/Notify）且被裁剪——确认包触发源改为命中事件广播（M7 分层） |
| 攻受两端感受不一致 | 各端本地表现参数不同步；或反馈依赖了仅单端存在的状态 |
