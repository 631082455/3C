# 移动与手感（Character / Movement）

覆盖：速度模型、加速度与转向、方向性手感、位移技能、重量感分级。

## 1. 速度模型：先定乘区，再定数值

移动速度的修改来源在实时战斗游戏里会迅速膨胀（buff/debuff、技能自带、地形、姿态）。数值设计前先定**乘区结构**，例如：

```
最终速度 = 基础速度 × 方向性系数 × Π(增益乘区) × Π(减益乘区) × 强制覆盖(如有)
```

- 增益/减益分开乘区并各设上下限（如减速合计不低于 35%），避免叠加失控。
- **禁止任何来源直接 set 速度**（绕过乘区 = 制造 M3 的 last-wins 问题）。
- 强制覆盖（如固定速度的冲锋）作为最高层显式存在，带完整生命周期（M4）。

## 2. 加速度与转向：手感的真正来源

同样的最高速度，加速/转向参数不同，手感天差地别。核心参数组：

| 参数 | 手感影响 | 典型取向 |
|---|---|---|
| 加速度（到达最高速的时间） | 响应性 vs 重量感 | 竞技向：80~150ms 内到 90% 速度；重量感角色可 2~3 倍 |
| 减速度/摩擦（松键停下的时间） | 精确停位 vs 滑行感 | 通常比加速更快（急停干脆） |
| 转向速率（180° 掉头耗时） | 灵活 vs 笨重，重量分级的主参数 | 轻甲与重甲拉开 1.5~2 倍差距 |
| 转向时是否保速 | 掉头的顺滑度 | 保速顺滑、降速厚重 |

**重量感分级**建议做成移动预设（轻/中/重三档打包上述参数），新角色选档 + 微调（M9）。

## 3. 方向性手感

前进/侧移/后退给不同速度系数（如前 1.0 / 侧 0.9 / 后 0.8），是低成本高收益的手感分层手段：

- 强化"面向"的战术意义（走 A、风筝、逼近的博弈感）。
- 系数默认 1.0（不改变现状），逐角色/逐模式开启——灰度铺开，不做全局硬切。
- **方向判定必须用意图量（输入/加速度方向 vs 面向），不能用 Velocity**（M11）：被击退时 Velocity 朝后，若按结果量判定会把击退也打折，且预测端与权威端可能判断不一致。

## 4. 位移技能设计清单

每个位移技能（闪现/冲刺/跳跃/拉拽）过一遍：

1. **轨迹类型**：瞬移 / 匀速直线 / 曲线（加速度包络）/ 抛物线。轨迹决定镜头跟随策略（见 `camera-design.md` §4）。
2. **可打断性**：位移中能否被控制打断？打断后落在哪（当前点/回起点/最近合法点）？
3. **方向来源**：指针方向 / 移动输入方向 / 面向。三者不一致时用哪个？（快速施法下这是高频争议点）
4. **终点合法性**：撞墙/越障/进禁区的裁定规则；预览显示的落点必须与执行一致（M8）。
5. **位移中的输入**：能否转向？能否预输入下一动作（衔接普攻/技能的缓冲窗口）？
6. **速度残留**：位移结束瞬间速度归还给正常移动的方式——硬切到 0（干脆）还是带出口速度衰减（顺滑）。这是"位移手感"的隐藏关键参数。
7. **生命周期**：死亡/被打断时轨迹修饰、无敌帧、碰撞豁免的清理（M4）。

## 5. 动画与移动的配合

- **动画跟随移动，而不是移动跟随动画**：位移距离/速度由移动系统的参数驱动，动画用 root motion 时必须校准到与参数一致，或直接禁 root motion 改程序位移（网络预测项目强烈建议后者，见 `netcode-3c.md`）。
- 起步/急停/转身可加短过渡动画，但**过渡动画不得增加输入到位移的延迟**——表现慢可以，响应慢不行。先动起来，动画追。
- 战斗姿态（持武器）与常规姿态用叠加层（additive/overlay）切换，而非整套动画翻倍；姿态切换的进入/退出时机（进战/脱战判定）要与移动参数切换（如有）同步。

## 6. 常见移动手感问题 → 排查方向

| 症状 | 优先排查 |
|---|---|
| "飘" | 加速/减速太慢；转向保速过度；急停缺失 |
| "笨" | 输入到位移延迟（过渡动画阻塞？输入缓冲丢失?）；加速度过低 |
| 改了速度没效果 | 乘区里被别的来源盖了；生效链逐层验证（M10）；死配置（M14） |
| 偶现速度异常残留 | 某个速度修饰缺死亡/移除清理（M4）；有来源绕过乘区直接 set |
| 位移结束"顿一下" | 出口速度硬切 + 镜头 FOV 回收过快叠加；分开调 |
