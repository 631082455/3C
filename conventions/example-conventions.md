# 项目 3C 约定 — 《环山之战》（虚构示例项目）

> 这是一个**虚构的 5v5 MOBA 项目**示例，展示约定模板的填写深度。请勿直接使用。

## 1. 项目基本盘

- **游戏类型**：5v5 MOBA，第三人称视角（非传统俯角）
- **视角与镜头基础形态**：第三人称跟随镜头，玩家可小幅调整俯仰，无自由旋转对局模式
- **对局节奏**：团队对抗，单次小规模交战 5~15 秒，团战 15~40 秒
- **体验基调关键词**：
  - **干脆**：技能启动与位移瞬发感优先，宁可牺牲演出长度
  - **信息清晰**：镜头永远保证玩家角色与主要威胁同屏，构图让位于信息
  - **重量分级**：坦克与刺客的移动/打击手感必须一眼可辨
  - **操作权神圣**：任何系统夺取玩家镜头/移动控制权都需要极强的理由

## 2. 输入

- **主要输入设备**：键鼠为主，手柄适配为次
- **移动方式**：WASD 直接控制
- **施法框架**：默认快速施法，可选目标预览确认；带落点的技能一律"预览态 → 执行态"两段式
- **已有的全局输入规则**：全局输入缓冲 200ms；后摇取消队列深度 1；移动输入永远可打断非锁定后摇

## 3. 网络模型

- **是否有客户端预测**：是
- **预测路径上的系统**：移动（含移速修饰）、技能启动、位移技能轨迹
- **权威端**：专用服务器
- **已知禁忌**：预测路径上的任何逻辑禁止读取"结果量"（如 Velocity）判断意图，必须读"意图量"（如输入/加速度）；预测路径禁止依赖仅服务器存在的状态

## 4. 现有 3C 系统形态

### 4.1 移动
- 速度/加速度/转向逐英雄配置于英雄数据资产，运行时经统一乘区叠加 buff
- 已有方向性速度系数（前进/侧后方向独立系数），默认 1.0，逐英雄开启

### 4.2 镜头
- 自研镜头效果层：效果以资产形式配置（震屏/推拉/FOV/偏移），带优先级 Band 仲裁
- 优先级 Band 从低到高：Ambient < Feedback < Skill < Cinematic；同 Band 内按权重混合，跨 Band 高者覆盖
- 所有镜头效果必须声明结束条件与死亡清理行为，资产校验器强制检查

### 4.3 动画/表现
- Montage 分层：全身 / 上半身 / 叠加（Additive）槽；策划可配蒙太奇资产与 Notify
- 打击反馈已有件：命中顿帧（逐技能配置）、震屏（走镜头效果层）、受击方向性动画（4 向）、材质闪白（全局默认+可覆盖）

## 5. 制作与协作约定

- **配置体验要求**：新英雄 3C 接入 = 选移动预设（轻/中/重）+ 改 1~2 个数；出现"必须写代码才能配"的方案要在需求清单中显式标红
- **文档产出语言与格式**：中文 Markdown，评审版可转 HTML
- **评审关注点**：① 生命周期是否闭环（死亡/重生/打断）② 与现有乘区/Band 的兼容 ③ 手柄端是否同样成立

## 6. 硬性禁忌

- 任何效果不得直接写玩家镜头朝向（rotation）；旋转类演出只允许出现在玩家无操作权的状态（死亡/回放/过场）
- 移速修改必须走统一乘区，禁止直接 set 速度值
- 表现层（特效/Cue）禁止承载周期性结算逻辑
- 任何自我施加的控制效果（自晕/自缚）必须显式声明"不取消自身来源技能"
