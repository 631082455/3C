# 出库单 v3：工作会话直接产 H5 案例稿，维护会话守入库关

> 用途：任何一个 AI 工作会话做完有方法论价值的 3C 工作后，把下面的「H5 案例稿指令」
> 原样贴给它，让它**以母版为基底直接产出 H5 案例稿**，然后你和它在 H5 上迭代
> （改内容、出图、定制模块）——迭代发生在有一手上下文的地方。
> 知识库维护会话不参与逐轮迭代，只守**入库关**：主题合规、术语、M/AP 反哺、目录、发布。
>
> **三重定位**（缺一不可）：
> 1. **策划作品集**——展示资深 3C 管线负责人的决策力：体验出发、业界视野、方案排定；
> 2. **工程落地复盘**——展示真实落地能力：对齐、实现、实验、数据。可信度的来源；
> 3. **3C 管线知识库**——沉淀可复用规律，反哺元规则/反模式，供未来的工作与 agent 检索。

---

## H5 案例稿指令（原样粘贴给工作会话的 agent）

```
请把本次工作写成一篇 H5 案例稿。分两步：

【第一步：拿母版】
读取 C:\Users\User\3C\templates\新版式预览 - 战斗镜头动效层设计稿.html ——
这是本知识库定稿的「精装手册」视觉母版。保留它的全部 <style>、封面(hero)结构、
图签(docinfo)、粘性目录、亮暗切换与 scrollspy 脚本；替换全部正文内容为本案例。
产出写入：C:\Users\User\3C\knowledge-local\inbox\<今天日期>-<主题短名>.html
（迭代期都在这个文件上改；定稿入库由知识库维护会话执行。）

【第二步：按骨架写正文】
定位：战斗 3C 管线负责人的案例复盘。三类读者：评审同行（看决策过程）、
合作程序（看工程落地）、未来检索知识库的人和 agent（看可复用规律）。
封面图签填：文档编号 CASE-XX / 版本·状态 / 需求方·实现方 / 配套文档(如有)。
章节按以下骨架（可按本案例实际情况增删小节，但大骨架不动）：

1.【体验目标】要玩家感受到什么（玩家体验语言开篇，禁止技术开场），
   现状哪里背离——附玩家/测试者原始反馈
2.【问题定义】体验落差拆成明确矛盾：哪两个正确目标打架、为什么不能全都要、
   不解决的代价
3.【业界调研】逐案写"它的做法 → 借了什么 / 为什么排除"，注明核实方式
4.【方案排定】候选路径、取舍标准（体验优先级/成本/风险）、每步为什么。
   写决策过程，不要只给结论
5.【终案分层】分层拆解，每层一句"它守住了哪条体验"
6.【工程落地实录】与程序对齐的接口与约束、实现中暴露的新问题及应对、
   实验与证伪（时间序：假设→验证→数据→结论）。被否方案保留完整篇幅
7.【关键参数】拍板的数字/阈值/曲线，逐条附体验依据
8.【验收实录】实测结论与用户/玩家原话（注明原话）
9.【遗留与保险丝】挂着的需求、兜底设计、已知妥协
10.【方法论沉淀】可复用规律：候选元规则/反模式/检查清单，每条写适用边界
11.【附录：未入正文的素材】没进正文但可能有用的细节全倒进来，宁滥勿缺

【视觉与出图硬规则】（违反任何一条都算不合格）
- 只许使用母版已有的 CSS 变量与组件 class（.rule/.note/.grid/.box/.docinfo/
  tr.group/.ok/.no/.later 等）；需要新版式时用现有变量组合实现；
- 禁止引入新色值、渐变文字、外部字体、CDN、任何外部请求——单文件自包含；
- 示意图优先用内联 SVG，颜色一律取 CSS 变量（var(--accent)/var(--brass)/
  var(--line)/var(--muted)/var(--ink)），保证亮暗双主题下都成立；
  几何/分层/时序类内容能画图就画图，别用大段文字硬扛；
- 截图/录屏用本地相对路径，GIF 建议转 MP4 用 <video autoplay loop muted playsinline>。

【写作纪律】
- 每个决策写"为什么"而不只"是什么"；忠实优先于好看；证伪与失败必须保留；
- 禁营销腔（"我们成功地/完美地"）；结论用数据和原话支撑；记不清标注"存疑"。
```

## 入库关（知识库维护会话执行，迭代定稿后）

1. 主题合规检查：无新色值/外部依赖/禁项；图签、目录、亮暗切换完好；
2. 术语与既有文档对齐（只改形式不改事实）；
3. 反哺：对照第 10 节「方法论沉淀」，新增/并入 `references/` 的 M/AP 条目，
   案例稿与元规则建双向引用；
4. 移入 `docs/`（案例复盘目录），首页/总览加条目；
5. commit → push → Pages 生效；inbox 里的迭代稿留档（不入 git）。
