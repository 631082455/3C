# 出库单 v3.1：工作会话直接产 H5 案例稿，维护会话守入库关

> 用途：任何一个 AI 工作会话做完有方法论价值的 3C 工作后，把下面的「H5 案例稿指令」
> 原样贴给它（把文件名占位改成实际日期和主题），让它**以空白母版为基底填空产出
> H5 案例稿**，然后你和它在 H5 上迭代（改内容、出图、定制模块）——迭代发生在
> 有一手上下文的地方。
> 知识库维护会话不参与逐轮迭代，只守**入库关**：主题合规、术语、M/AP 反哺、
> CASE 编号分配、目录、发布。
>
> **三重定位**（缺一不可）：
> 1. **策划作品集**——展示资深 3C 管线负责人的决策力：体验出发、业界视野、方案排定；
> 2. **工程落地复盘**——展示真实落地能力：对齐、实现、实验、数据。可信度的来源；
> 3. **3C 管线知识库**——沉淀可复用规律，反哺元规则/反模式，供未来的工作与 agent 检索。

---

## H5 案例稿指令（原样粘贴给工作会话的 agent）

```
请把本次工作写成一篇 H5 案例稿。

【第一步：复制空白母版】
把 C:\Users\User\3C\templates\案例稿空白母版.html 复制为：
C:\Users\User\3C\knowledge-local\inbox\<今天日期>-<主题短名>.html
母版已预置：精装手册主题样式、封面与图签、11 段章节骨架、目录、亮暗切换。
你的工作是【填空】：替换所有〖【占位】〗文本与标题占位，按每个 <section> 内
HTML 注释的指引写正文。不许修改 <style>、脚本、章节骨架结构（章节内可自由
增设 h3 小节与组件）。图签的文档编号保持 CASE-XX 不动（入库时统一分配）。

【第二步：写正文】
定位：战斗 3C 管线负责人的案例复盘。三类读者：评审同行（看决策过程）、
合作程序（看工程落地）、未来检索知识库的人和 agent（看可复用规律）。
全文要同时回答：基于哪些体验？想解决什么问题？业界怎么做？方案怎么排定？
落地时真实发生了什么？沉淀出了什么可复用的东西？
各章要求以母版内各 section 的注释为准，其中：
- 【体验目标】必须用玩家体验语言开篇，禁止技术开场；
- 【方案排定】写决策过程（候选、取舍标准、每步为什么），不要只给结论；
- 【工程落地实录】被否掉的方案保留完整篇幅——"为什么它看起来对、实际错在哪"；
- 【方法论沉淀】每条规律写清适用边界（什么情况成立、什么情况不适用）。

【视觉与出图硬规则】
- 只用母版已有 CSS 变量与组件 class（.rule/.note/.grid/.box/table/tr.group/
  .ok/.no/.later 等），需要新版式用现有变量组合；
- 禁止新增色值、渐变文字；禁止加载外部资源（字体/脚本/图片 CDN）——单文件
  自包含。指向参考资料的普通超链接允许；
- 示意图优先内联 SVG（class="diagram"），颜色一律取 CSS 变量
  （var(--accent)/var(--brass)/var(--line)/var(--muted)/var(--ink)），
  保证亮暗双主题都成立；几何/分层/时序类内容能画图就画图，别用大段文字硬扛；
- 截图/录屏用本地相对路径，GIF 建议转 MP4 用 <video autoplay loop muted playsinline>。

【写作纪律】
- 每个决策写"为什么"而不只"是什么"；忠实优先于好看；证伪与失败必须保留；
- 禁营销腔（"我们成功地/完美地"）；结论用数据和原话支撑；记不清标注"存疑"。

【交稿前自查】（逐条确认后才算完成）
1. 全文搜索「占位」「〖」——必须零残留；
2. 除 CASE-XX 外无母版遗留的示例文本；
3. 目录 11 项锚点与正文 section id 一一对应；
4. 全文无新增 16 进制色值（允许的只有母版 <style> 里原有的）、无外部资源加载；
5. 所有 SVG 填色/描边都用 var(--*)，没有写死颜色。
```

## 入库关（知识库维护会话执行，迭代定稿后）

1. 主题合规检查：无新色值/外部资源加载/禁项；图签、目录、亮暗切换完好；
2. 分配 CASE 编号（替换 CASE-XX，按 docs/ 内既有案例递增）；
3. 术语与既有文档对齐（只改形式不改事实）；
4. 反哺：对照第 10 节「方法论沉淀」，新增/并入 `references/` 的 M/AP 条目，
   案例稿与元规则建双向引用；
5. 移入 `docs/`（案例复盘目录），首页/总览加条目；
6. commit → push → Pages 生效；inbox 里的迭代稿留档（不入 git）。
