# 参数表模板

> 两段式：先预设表（策划日常用），后原子参数表（预设制作者用）。参数 > 5 个而没有预设表 = 不合格（M9）。

## 预设表

〈预设是给"日常配置"用的。命名用体验词（轻快/标准/厚重），不用数值词。〉

| 预设 | 适用对象 | 体验描述 | 关键差异参数 |
|---|---|---|---|
| 轻快 | 〈刺客/敏捷型〉 | 〈一句话手感描述〉 | 〈只列与"标准"不同的 2~4 项〉 |
| 标准 | 〈默认〉 | | （基准） |
| 厚重 | 〈坦克/重型〉 | | |

**日常工作流**：选预设 → 允许微调的参数仅限：〈列 1~3 个白名单参数〉。改白名单以外的参数需评审。

## 原子参数表

〈每参数一行，五列缺一不可。生效链格式：配置位置 → 读取系统 → 应用点（M10）。〉

| 参数 | 默认值 | 可调范围 | 调整后果（大↑ / 小↓） | 生效链 |
|---|---|---|---|---|
| 〈参数名〉 | | | ↑：〈会怎样〉 ↓：〈会怎样〉 | 〈资产/表〉→〈系统〉→〈作用点〉 |

## 标注约定

- **[预测]**：该参数在客户端预测路径上，改动需两端一致生效（见 `netcode-3c.md` §5）
- **[平衡]**：该参数影响竞技平衡（如前摇锁定时长），改动走平衡评审而非手感评审
- **[红]**：现有系统配不出来，属程序需求
