# 出库单：工作会话 → 知识库素材包

> 用途：任何一个 AI 工作会话（不限工具）做完一件有方法论价值的 3C 工作后，
> 把下面的「提取指令」原样贴给它，让它产出一份**素材包**落到本地中转区。
> 素材包不是成品——入库（脱敏、成稿、配主题、反哺元规则）由知识库维护会话完成。

---

## 提取指令（原样粘贴给工作会话的 agent）

```
请把本次工作整理成一份「知识库素材包」，纯 Markdown，写入：
C:\Users\User\3C\knowledge-local\inbox\<今天日期>-<主题短名>.md
（该目录不入 git，放心写全细节。）

只做忠实记录：不要美化、不要写成文章、不要做任何 HTML/排版。按以下结构输出：

1. 【矛盾】一句话说清要解决的体验矛盾（例：镜头演绎会挪准心）
2. 【验收标准】当时定下的"做完"判据，尽量用原话
3. 【探索时间线】按真实顺序列出：每个假设 → 做了什么 → 结果（证实/证伪）。
   被否掉的方案和否掉的理由必须写全——这是素材包里最值钱的部分。
4. 【终案结构】最终方案的分层拆解，每层附一句"它防住了什么问题"
5. 【关键参数】所有拍板的数字、阈值、曲线设定，以及各自的依据
6. 【调研谱系】参考过的业界案例，每个一句"从它那里借了什么/排除了什么"
7. 【验收实录】实测结论与用户/玩家原话引用
8. 【遗留清单】还挂着的需求、保险丝设计、已知的妥协
```

---

## 入库侧流程（知识库维护会话执行）

1. 读 `knowledge-local/inbox/` 中的素材包；
2. 判断产物形态（可多选）：
   - **案例稿** → `output/examples/`（方法探索 + 工程落地的完整叙事）
   - **元规则/反模式反哺** → `references/`（新增 M/AP 条目或并入既有条目）
   - **网页版** → `docs/`（精装手册主题 H5）
3. 交叉引用：链接相关 M/AP 条目与既有文档；
4. commit → push → Pages 生效；素材包原件留在 inbox 归档（不入 git）。
